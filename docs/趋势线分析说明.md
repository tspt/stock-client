# 趋势线分析说明文档

## 概述

趋势线分析用于在**最近 M 根 K 线构成的检索窗口**内，判断是否存在**连续 N 根**满足「沿趋势上行」的约束：每一根收盘价不跌破前一日收盘，且收盘价站在当日 **MA5** 之上（含数值容差）。  
实现代码位于 `src/utils/trendLineAnalysis.ts` 中的 `calculateTrendLineInLookback`。

---

## 核心判定规则

对窗口内某一连续区间，起点下标为 `i`、长度为 `N`（覆盖下标 `i … i+N-1`），当且仅当对区间内每一根 `j` 均满足：

1. **收盘不跌**：`close[j] ≥ close[j-1]`（实现上使用微小 `EPSILON`，避免浮点误差导致边界误判）。
2. **站在 MA5 之上**：`close[j] ≥ MA5(j)`，其中 `MA5(j)` 为截至第 `j` 根收盘的 5 日简单移动平均（前四根因样本不足不参与趋势线判定）。

若存在多组合法起点，取**起点下标最大**的一组，即**最靠近最新 K 线**的一段（贴近当前走势）。

---

## 参数说明

### 1. 检索根数 M（lookback）

**含义**：从最新一根 K 线向前，最多考察多少根 K 线构成的窗口。

**关系**：实际窗口长度 `m = min(M, 当前 K 线总根数)`，窗口起点为 `len - m`。

**页面默认值**：机会分析页筛选区可与服务层默认值不同；用户修改后，会基于本地缓存的 K 线**重新计算** `trendLine` 字段。

---

### 2. 连续根数 N（consecutive）

**含义**：上述窗口内需要连续满足的 K 线根数。

**约束**：界面中通常要求 `N ≤ M`（且为正整数）。

---

## 结果字段（`TrendLineAnalysis`）

| 字段 | 含义 |
|------|------|
| `lookback` | 实际参与检索的窗口长度 `m` |
| `consecutive` | 连续根数 `N` |
| `isHit` | 是否在窗口内找到满足条件的连续 `N` 根 |
| `reasonText` | 列表/导出用的简短说明，例如含「含尾 / 非尾」（是否贴到最新一根为区间右端）、「检 m 无 N 连」、「数据不足」等 |

**含尾 / 非尾**：当命中段的起点为 `len - N` 时，该连续区间一直延伸到最新一根 K 线，说明中标记为「含尾」；否则为「非尾」（命中段落在窗口内但最新一根不在该连续上涨段末尾）。

---

## 在应用中的数据流

1. **批量分析（`opportunityService.analyzeOneStock`）**  
   拉取 K 线后调用 `calculateTrendLineInLookback`，使用服务内写死的 `lookback` / `consecutive`（与页面初始默认一致），写入 `StockOpportunityData.trendLine`。

2. **机会分析页（`OpportunityPage`）**  
   使用 `klineDataCache` 与用户在筛选区设置的 M、N **再次计算**趋势线，保证表格「趋势线 / 趋势线说明」列与筛选参数一致。

3. **列表筛选**  
   启用「趋势线筛选」时，仅保留 `trendLine.isHit === true` 的条目；该判断为 O(1)，成本主要在事前计算好的 `trendLine` 上。

4. **与横盘筛选的关系**  
   可同时开启，逻辑为 **AND**（需同时满足各自条件）。

---

## 实现说明（与语义等价）

为保证性能与可维护性，实现上采用与定义等价的写法：

1. **MA5 预计算**：对整段 K 线一次性构建每根收盘对应的 MA5 数组，避免在每个候选起点、每一段内重复做重叠窗口求和。
2. **从右向左枚举起点**：因目标为**最大的合法起点 `i`**，可从 `i = len - N` 向下扫描至窗口左端，**首次命中即结束**，与从左向右扫完全窗口取最大 `i` 的结果一致。

修改算法后，应对同一组 K 线与同一组 M、N 得到相同的 `isHit` 与 `reasonText`（回归时可固定样本对照）。

## 自检清单（代码审阅 / 回归）

- [ ] **数据不足**：K 线根数 `< N` 时返回「数据不足」，且不抛错。
- [ ] **无命中**：窗口内不存在连续 N 根满足条件时，`isHit` 为 false，`reasonText` 含「检 m 无 N 连」。
- [ ] **含尾**：当且仅当命中起点为 `len - N` 时，说明中含「含尾」。
- [ ] **与横盘 AND**：同时开启横盘 + 趋势线时，仅显示两类条件均满足的标的。
- [ ] **构建**：`npx vite build` 可通过（前端产物包含本模块）。
