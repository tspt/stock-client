#### 1. AI 时间衰减后未使用有效值

**文件**: `src/workers/opportunityFilterWorker.ts`

**问题描述**:

- 第 511-516 行计算了`effectiveConfidence`（应用时间衰减后的置信度）
- 但第 337-362 行的置信度范围检查仍使用原始值`trendPrediction.confidence`
- 导致开启时间衰减功能后，筛选逻辑矛盾

**修复方案**:

```typescript
// 将时间衰减计算提前到所有置信度检查之前
let effectiveConfidence = trendPrediction?.confidence || 0;
if (filters.aiEnableTimeDecay && item.analysisTimestamp) {
  const hoursPassed = (Date.now() - item.analysisTimestamp) / (1000 * 3600);
  const decayRate = filters.aiDecayRate || 0.1;
  effectiveConfidence = effectiveConfidence * Math.exp(-decayRate * hoursPassed);
}

// 后续所有置信度检查都使用effectiveConfidence
const confidencePercent = effectiveConfidence * 100;
```

**影响**: 修复后，时间衰减功能正常工作，旧分析的置信度会随时间降低

#### 2. MACD 背离检测逻辑错误

**文件**: `src/utils/technicalIndicators.ts`

**问题描述**:

- 原实现只查找"全局最高点/最低点"，而非"局部波峰/波谷"
- 无法正确识别真正的顶背离/底背离形态
- 只能检测到极端情况，漏判率高

**修复方案**:

```typescript
// 新增局部极值点检测函数
function findLocalPeaks(data: number[], windowSize: number): number[] {
  const peaks: number[] = [];
  for (let i = windowSize; i < data.length - windowSize; i++) {
    let isPeak = true;
    for (let j = i - windowSize; j <= i + windowSize; j++) {
      if (j !== i && data[j] >= data[i]) {
        isPeak = false;
        break;
      }
    }
    if (isPeak) peaks.push(i);
  }
  return peaks;
}

// 使用最近的两个波峰/波谷判断背离
const pricePeaks = findLocalPeaks(windowCloses, 2).slice(-2);
const difPeaks = findLocalPeaks(windowDif, 2).slice(-2);

// 顶背离：价格第二个峰更高，但DIF第二个峰更低
if (pricePeaks.length >= 2 && difPeaks.length >= 2) {
  const priceHigher = windowCloses[pricePeaks[1]] > windowCloses[pricePeaks[0]];
  const difLower = windowDif[difPeaks[1]] < windowDif[difPeaks[0]];
  if (priceHigher && difLower) return true;
}
```

**影响**: MACD 背离检测准确率大幅提升，能正确识别技术分析中的经典背离形态

### 🟡 P1 级别 - 重要优化（3 个）

#### 3. 趋势判断阈值过小

**文件**: `src/utils/trendPatterns.ts`

**问题描述**:

- 原阈值 0.1%（`avgPrice * 0.001`）
- 对于 100 元股票，20 天涨 1 元就判定为"上升趋势"
- 几乎等同于横盘，误判率极高

**修复方案**:

```typescript
// 提高到0.5%
return slope > avgPrice * 0.005; // 上升
return slope < -avgPrice * 0.005; // 下降
```

**影响**: 趋势判断更准确，减少假信号

#### 4. 突破/跌破形态无成交量确认

**文件**: `src/utils/trendPatterns.ts`

**问题描述**:

- 原实现只检查价格突破，不验证成交量
- 无法过滤"无量假突破"
- 实战中假突破占比高

**修复方案**:

```typescript
export function isBreakout(klineData, lookback = 20, requireVolume = true) {
  // ... 价格突破检查 ...

  // 成交量确认：当日成交量 > 前20日均量 * 1.5
  if (requireVolume) {
    const avgVolume = recent.reduce((sum, k) => sum + k.volume, 0) / recent.length;
    if (lastKline.volume < avgVolume * 1.5) {

    }
  }

  return true;
}
```

**影响**: 有效过滤假突破/假跌破，提高信号可靠性

#### 5. 横盘强度计算公式缺陷

**文件**: `src/utils/consolidationAnalysis.ts`

**问题描述**:

- 原公式：`100 - (metric/threshold)*100`
- 线性计算导致边界突变：metric=1.4 时 strength=6.7，metric=1.5 时 strength=0
- 强度分布不均匀，排序效果差

**修复方案**:

```typescript
// 改用指数衰减
function calculateStrength(metric: number, threshold: number): number {
  if (threshold <= 0 || metric >= threshold) return 0;
  const ratio = metric / threshold;
  return round(Math.exp(-3 * ratio) * 100);
}
```

**效果对比**:
| metric/threshold | 原公式 | 新公式 |
|-----------------|--------|--------|
| 0.0 | 100 | 100 |
| 0.3 | 70 | 40.7 |
| 0.5 | 50 | 22.3 |
| 0.8 | 20 | 9.1 |
| 1.0 | 0 | 5.0 |

**影响**: 强度评分更平滑，横盘质量排序更合理

#### 8. 添加筛选条件互斥规则提示

**新建文件**: `src/utils/filterCompatibility.ts`

**功能**:

1. **兼容性检查**: 检测逻辑冲突的筛选组合

   - 横盘 vs 上升/下降趋势
   - 仅急跌 vs AI 看涨
   - 突破 vs 横盘
   - RSI 超买/超卖与趋势方向矛盾

2. **复杂度评分**: 评估筛选条件的计算复杂度（0-100 分）

   - 技术指标：5-10 分/项
   - K 线形态：3 分/项
   - 横盘/趋势线分析：12-15 分
   - AI 分析：20 分+

3. **性能提示**: 根据复杂度给出速度预估
   - <20 分: ✅ 速度快
   - 20-50 分: 💡 可能需要几秒
   - 50-80 分: ⚠️ 可能需要较长时间
   - > 80 分: 🔴 建议减少筛选项

**使用示例**:

```typescript
import {
  checkFilterCompatibility,
  getFilterComplexityScore,
  getPerformanceHint,
} from '@/utils/filterCompatibility';

const warnings = checkFilterCompatibility(filters);
if (warnings.length > 0) {
  warnings.forEach((w) => message.warning(w));
}

const score = getFilterComplexityScore(filters);
message.info(getPerformanceHint(score));
```
