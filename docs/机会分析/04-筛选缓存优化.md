# 筛选条件缓存优化

## 优化背景

**问题1**: 原实现在点击“一键分析”时保存筛选条件，会导致：
1. 需要重新查询API获取数据，速度慢
2. 用户体验差，每次分析都要等待

**问题2**: 切换路由后再回来，筛选条件恢复为默认值
- 原因：自动保存的useEffect在组件挂载时立即执行
- 在恢复逻辑完成前就保存了默认值，覆盖了localStorage中的正确值

**解决方案**: 
- 移除“一键分析”时的保存逻辑
- 改为筛选条件变化时自动保存（防抖300ms）
- **添加恢复标记，确保在恢复完成前不触发自动保存**
- 页面卸载时也保存（兜底机制）

## 优化内容

### 1. 移除"一键分析"中的保存逻辑

**优化前**:
```typescript
const handleAnalyze = async () => {
  // ... 构建prefs对象（76行代码）
  const prefs: OpportunityFilterPrefs = { /* ... */ };
  saveOpportunityFilterPrefs(prefs); // ❌ 这里会触发不必要的保存
  
  await startAnalysis(currentPeriod, filteredStocks, currentCount);
};
```

**优化后**:
```typescript
const handleAnalyze = async () => {
  if (filteredStocks.length === 0) {
    message.warning('当前市场暂无股票数据');
    return;
  }

  // 直接开始分析，不需要保存筛选条件（已通过useEffect自动保存）
  await startAnalysis(currentPeriod, filteredStocks, currentCount);
};
```

### 2. 添加筛选条件变化时自动保存

新增一个useEffect，监听所有筛选条件的变化，使用防抖机制自动保存：

```typescript
// 筛选条件变化时自动保存（防抖300ms）
useEffect(() => {
  const timer = setTimeout(() => {
    const prefs: OpportunityFilterPrefs = {
      version: 1,
      selectedMarket,
      nameType,
      // ... 所有筛选条件
    };
    saveOpportunityFilterPrefs(prefs);
  }, 300); // 防抖300ms

  return () => clearTimeout(timer);
}, [
  // 所有筛选条件的依赖
  selectedMarket,
  nameType,
  priceRange,
  aiAnalysisEnabled,
  // ... 等等
]);
```

### 3. 保留页面卸载时的保存（兜底机制）

原有的页面卸载时保存逻辑保持不变，作为兜底机制：

```typescript
// 在页面卸载时保存当前的筛选条件
useEffect(() => {
  return () => {
    const prefs: OpportunityFilterPrefs = { /* ... */ };
    saveOpportunityFilterPrefs(prefs);
  };
}, [/* 所有依赖 */]);
```

## 技术细节

### 恢复标记机制

**问题**: 组件挂载时，自动保存的useEffect会在恢复逻辑完成前执行，导致默认值覆盖localStorage中的正确值。

**解决**: 使用ref标记恢复状态

```typescript
// 标记是否已完成初始恢复
const isRestoredRef = useRef(false);

// 恢复逻辑完成后设置标记
useEffect(() => {
  const hydrate = async () => {
    await loadCachedData();
    const prefs = loadOpportunityFilterPrefs();
    if (prefs) {
      applyOpportunityFilterPrefsToState(prefs, actions);
    }
    // 标记恢复完成
    isRestoredRef.current = true;
  };
  void hydrate();
}, [loadCachedData]);

// 自动保存时检查标记
useEffect(() => {
  // 如果还未完成初始恢复，不执行保存
  if (!isRestoredRef.current) {
    return;
  }
  
  const timer = setTimeout(() => {
    saveOpportunityFilterPrefs(prefs);
  }, 300);
  return () => clearTimeout(timer);
}, [/* 所有依赖 */]);
```

**工作流程**:
1. 组件挂载 → state初始化为默认值
2. 恢复useEffect执行 → 从localStorage读取并恢复state
3. 恢复完成 → 设置`isRestoredRef.current = true`
4. 之后state变化 → 自动保存useEffect才会执行

### 防抖机制

```typescript
const timer = setTimeout(() => {
  saveOpportunityFilterPrefs(prefs);
}, 300);

return () => clearTimeout(timer); // 清理函数
```

**作用**:
- 用户快速调整多个筛选条件时，只在停止操作300ms后保存一次
- 避免频繁的localStorage写入操作
- 提升性能，减少不必要的I/O

### 依赖数组

useEffect的依赖数组包含所有筛选条件状态：
- 查询条件：selectedMarket, nameType, currentPeriod, currentCount
- 数据筛选：priceRange, marketCapRange, turnoverRateRange等
- 横盘筛选：consolidationTypes, consolidationLookback等
- AI筛选：aiAnalysisEnabled, aiTrendUp等
- UI状态：filterPanelActiveKey

**任何筛选条件变化都会触发自动保存**。

## 注意事项

1. **防抖时间**: 当前设置为300ms，可根据实际需求调整
   - 太短：可能频繁写入
   - 太长：用户感觉延迟

2. **依赖管理**: 确保所有筛选条件都加入依赖数组
   - 遗漏会导致某些条件变化时不保存
   - 多余的依赖会导致不必要的保存

3. **错误处理**: saveOpportunityFilterPrefs已有try-catch
   - localStorage满时会捕获异常
   - 不会影响正常使用
