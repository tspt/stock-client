# AI 分析版本说明

## 版本对比

### v1.0 原始版

**特点：**

- 简单固定权重算法
- 统一阈值，不考虑个股差异
- 6 类技术指标
- 计算速度快

**适用场景：**

- 交易新手
- 高频交易（速度优先）
- 快速原型验证

---

### v2.0 优化版

**新特性：**

- ✅ **市场状态检测**：识别高/中/低波动市场
- ✅ **自适应阈值**：基于个股历史数据计算个性化阈值
- ✅ **动态权重调整**：根据市场状态调整指标权重
- ✅ **信号持续性检测**：分析趋势的持续天数
- ✅ **成交量异常检测优化**：基于个股统计的阈值

**技术改进：**

```typescript
// v1.0: 固定阈值
if (lastRSI6 < 30) { ... }

// v2.0: 自适应阈值
const oversoldThreshold = characteristics?.rsiPercentile25 ?? 30;
if (lastRSI6 < oversoldThreshold) { ... }
```

**适用场景：**

- 有经验交易者
- 中长线投资（准确性优先）
- 震荡市（减少假信号）

### v3.0 增强版 ⭐ NEW

**基于 v2.0，新增筛选功能：**

#### 1. 信号共识筛选

**原理：** 要求多个技术指标同时指向同一方向

**新增筛选条件：**

- `aiSignalConfluence`: 是否要求信号共识
- `aiMinSignalCount`: 最少支持信号数量（默认 4）
- `aiMinSignalRatio`: 最小信号比例（默认 0.6 即 60%）

**价值：**

- ✅ 过滤掉指标矛盾的模糊信号
- ✅ 提高筛选结果的可靠性
- ✅ 已有数据，无需额外计算

**示例：**

```
股票A有7个指标发出信号：
- 4个看涨（MA、MACD、KDJ、RSI）
- 2个看跌（布林带、成交量）
- 1个横盘（趋势形态）

如果设置 aiMinSignalCount=4, aiMinSignalRatio=0.6
→ signalRatio = 4/7 = 57% < 60%
→ 不通过筛选（信号不够一致）
```

#### 2. 相似形态胜率筛选

**原理：** 基于历史上相似形态的股票后续表现进行筛选

**新增筛选条件：**

- `aiPatternWinRateRange`: 相似形态胜率范围（0-100%）
- `aiMinSimilarPatterns`: 最少相似股票数量（默认 3）

**价值：**

- ✅ 真正启用相似形态识别功能
- ✅ 提供统计学支持
- ✅ 基于历史数据验证形态有效性

**示例：**

```
股票A找到5个相似形态的股票：
B: +5%, C: +3%, D: -2%, E: +8%, F: +4%
胜率 = 4/5 = 80%

如果设置 aiPatternWinRateRange.min = 70
→ 80% ≥ 70%
→ 通过筛选
```

#### 3. 风险收益比筛选

**原理：** 确保潜在收益与潜在风险的比例合理

**新增筛选条件：**

- `aiMinRiskRewardRatio`: 最小风险收益比（默认 2.0）

**价值：**

- ✅ 专业交易者常用指标
- ✅ 避免高风险低收益的交易
- ✅ 确保盈亏比合理

**示例：**

```
支撑位：10元
阻力位：15元
当前价：11元

潜在收益 = 15 - 11 = 4元
潜在风险 = 11 - 10 = 1元
风险收益比 = 4/1 = 4.0

如果设置 aiMinRiskRewardRatio = 2.0
→ 4.0 ≥ 2.0
→ 通过筛选
```

## 使用建议

### v3.0 推荐配置

**保守型（高准确性）：**

```
✅ 信号共识：是
   - 最少信号数：5
   - 最小比例：0.7 (70%)
✅ 相似形态胜率：60% ~ 100%
   - 最少相似股票：3
✅ 风险收益比：≥ 2.5
```

**平衡型（兼顾数量和准确性）：**

```
✅ 信号共识：是
   - 最少信号数：4
   - 最小比例：0.6 (60%)
✅ 相似形态胜率：50% ~ 100%
   - 最少相似股票：2
✅ 风险收益比：≥ 2.0
```

**激进型（更多机会）：**

```
❌ 信号共识：否
❌ 相似形态胜率：未启用
❌ 风险收益比：未启用
```
