## ⚙️ 筛选引擎

### 两阶段过滤

为了优化性能，采用**两阶段过滤**策略：

#### 阶段 1：主线程轻量过滤

**过滤条件**（仅依赖 `StockOpportunityData` 字段）：

- 价格
- 市值范围
- 换手率范围
- 市盈率范围
- KDJ-J 范围
- 涨跌停统计

**优势**：

- ✅ 无需访问 K 线数据
- ✅ 快速缩小候选集
- ✅ 减少 Worker 负载

#### 阶段 2：Worker 复杂计算

**计算内容**（依赖 K 线数据）：

- 横盘检测
- 趋势线分析
- 单日异动模式
- RSI/MACD/布林带
- K 线形态识别
- 趋势形态识别

**优势**：

- ✅ 后台线程，不阻塞 UI
- ✅ 并行计算，性能好
- ✅ 智能重算，只计算激活的指标

### 防抖机制

```typescript
// 筛选条件变化时，300ms 后才触发重算
useDebounce(() => {
  triggerFilter();
}, 300);
```

**目的**：

- 避免用户快速调整参数时频繁计算
- 提升用户体验

### 任务取消

```typescript
let latestRequestId = 0;

function runFilterTask(requestId: number) {
  if (requestId <= cancelledThroughRequestId) {
    return; // 任务已过期，丢弃结果
  }
  // ... 执行筛选
}
```

**目的**：

- 保证结果一致性
- 避免旧任务覆盖新结果

## 🔢 技术指标实现

### RSI（相对强弱指数）

**文件**：`src/utils/technicalIndicators.ts`

**算法**：Wilder 平滑法

```typescript
export function calculateRSI(klineData: KLineData[], period: number = 14): (number | null)[] {
  // 1. 计算价格变化序列
  // 2. 分别计算上涨日和下跌日的平均幅度
  // 3. RS = 平均上涨幅度 / 平均下跌幅度
  // 4. RSI = 100 - (100 / (1 + RS))
}
```

**特点**：

- ✅ 可配置周期（6/12/24 日）
- ✅ 默认 6 日（与同花顺一致）
- ✅ 数据要求：`period + 1` 根 K 线

---

### MACD（平滑异同移动平均线）

**文件**：`src/utils/technicalIndicators.ts`

**计算公式**：

```
EMA(12) = 12日指数移动平均
EMA(26) = 26日指数移动平均
DIF = EMA(12) - EMA(26)
DEA = DIF的9日EMA
MACD柱 = (DIF - DEA) × 2
```

**检测方法**：

```typescript
// 金叉：DIF从下向上穿越DEA
isMACDGoldenCross(dif, dea, index);

// 死叉：DIF从上向下穿越DEA
isMACDDeathCross(dif, dea, index);

// 背离：价格创新高/新低但DIF未同步
hasMACDDivergence(klineData, dif, lookback);
```

**数据要求**：至少 35 根 K 线（26 + 9）

---

### 布林带（Bollinger Bands）

**文件**：`src/utils/technicalIndicators.ts`

**计算公式**：

```
中轨 = SMA(20)
标准差 = StdDev(20)
上轨 = 中轨 + 2 × 标准差
下轨 = 中轨 - 2 × 标准差
```

**位置判断**：

```typescript
// 上轨附近：收盘价 ≥ 上轨 × (1 - 5%)
isNearUpperBand(close, upper, middle, lower, (threshold = 0.05));

// 中轨附近：|收盘价 - 中轨| ≤ 带宽 × 5%
isNearMiddleBand(close, upper, middle, lower, (threshold = 0.05));

// 下轨附近：收盘价 ≤ 下轨 × (1 + 5%)
isNearLowerBand(close, upper, middle, lower, (threshold = 0.05));
```

**数据要求**：至少 20 根 K 线

## 📈 趋势形态识别

### 实现文件

`src/utils/trendPatterns.ts`

### 5 种趋势形态

#### 1. 上升趋势（Uptrend）

**算法**：线性回归斜率

```typescript
const slope = (n * sumXY - sumX * sumY) / (n * sumX2 - sumX * sumX);
return slope > avgPrice * 0.001; // 斜率 > 平均价格的 0.1%
```

#### 2. 下降趋势（Downtrend）

```typescript
return slope < -avgPrice * 0.001; // 斜率 < -平均价格的 0.1%
```

#### 3. 横盘整理（Sideways）

```typescript
const volatility = ((maxHigh - minLow) / minLow) * 100;
return volatility < 5; // 波动 < 5%
```

#### 4. 突破形态（Breakout）

```typescript
const prevHigh = Math.max(...previousKlines.map((k) => k.high));
return lastClose > prevHigh * 1.02 && isBullish; // 突破前高 2% 且收阳
```

#### 5. 跌破形态（Breakdown）

```typescript
const prevLow = Math.min(...previousKlines.map((k) => k.low));
return lastClose < prevLow * 0.98 && isBearish; // 跌破前低 2% 且收阴
```

**回溯窗口**：默认 20 根，可配置 5-200 根

## 🔄 横盘检测算法

### 实现文件

`src/utils/consolidationAnalysis.ts`

### 两种判断方法

#### 方法 1：价格波动率法

```typescript
波动率 = (最高价 - 最低价) / 平均收盘价 × 100%
是横盘 = 波动率 < 阈值(默认2%)
```

#### 方法 2：MA 收敛法

```typescript
MA离散度 = max(|MA5-平均MA|, |MA10-平均MA|, |MA20-平均MA|) / 平均MA × 100%
是横盘 = MA离散度 < 阈值(默认3%)
```

### 横盘强度评分

```typescript
价格波动率法强度 = 100 - (波动率 / 阈值) × 100
MA收敛法强度 = 100 - (MA离散度 / 阈值) × 100
综合强度 = (价格波动率法强度 + MA收敛法强度) / 2
```

**强度分级**：

- 0-30：弱横盘
- 30-60：中等横盘
- 60-80：强横盘
- 80-100：极强横盘

### MA10 约束

可选条件：连续 N 根横盘段内，每日收盘价 ≥ 当日 MA10

**用途**：确保横盘期间股价站稳均线，提高可靠性

## 📊 趋势线分析

### 实现文件

`src/utils/trendLineAnalysis.ts`

### 判定规则

在窗口内寻找连续 N 根 K 线，每根需满足：

1. ✅ `close[i] ≥ close[i-1]`（不跌）
2. ✅ `close[i] ≥ MA5(i)`（站在 MA5 之上）

### 优化算法

**从右向左扫描**：

```typescript
for (let i = len - N; i >= windowStart; i--) {
  if (checkConsecutive(i, N)) {
    return { isHit: true, startIndex: i }; // 找到最大的起点即返回
  }
}
```

**优势**：

- ✅ 首次命中即结束，无需遍历全部
- ✅ 自然得到最靠近最新的命中段

### MA5 预计算

```typescript
// 一次性构建每根收盘对应的 MA5 数组
const ma5Array = buildMA5Array(klineData);
```

**优势**：避免在每个候选起点重复计算重叠窗口

## 🚀 性能优化

### 1. Web Worker 并行计算

**优势**：

- ✅ 不阻塞主线程
- ✅ 充分利用多核 CPU
- ✅ UI 保持流畅

### 2. 智能重算

```typescript
// 只在相关筛选条件激活时才计算对应指标
if (filters.rsiRange.min !== undefined || filters.rsiRange.max !== undefined) {
  const rsi = calculateRSI(klineData, filters.rsiPeriod);
  // ...
}
```

**效果**：减少 60-80% 的不必要计算

### 3. K 线数据增量同步

```typescript
// 全量同步（变化比例 > 30%）
worker.postMessage({ type: 'set-data-full', data: klineDataMap });

// 增量同步（变化比例 ≤ 30%）
worker.postMessage({
  type: 'set-data-patch',
  patches: { upsert: [...], remove: [...] }
});
```

**效果**：减少 70-90% 的数据传输

### 4. 条数上限控制

```typescript
const MAX_OPPORTUNITY_KLINE_CACHE_ENTRIES = 500;

if (analysisData.length > MAX_OPPORTUNITY_KLINE_CACHE_ENTRIES) {
  // 优先保留 K 线缓存命中的标的
  analysisData = prioritizeCachedEntries(analysisData, klineDataCache);
}
```

**目的**：避免 Worker 处理过多数据导致内存溢出

### 5. 预计算优化

**K 线形态检测**：

- 预计算每根 K 线的实体、影线、范围等
- 避免重复计算，提速 3-5 倍

**趋势线分析**：

- 预计算 MA5 数组
- 从右向左扫描，首次命中即返回

#### 2. 核心筛选函数

```typescript
function passesAIFilter(item: StockOpportunityData, filters: OpportunityFilterSnapshot): boolean {
  // 如果未启用AI筛选，直接通过
  if (!filters.aiAnalysisEnabled) {
    return true;
  }

  // 如果没有AI分析数据，不通过
  if (!item.aiAnalysis) {
    return false;
  }

  const { trendPrediction, recommendation, similarPatterns } = item.aiAnalysis;

  // 1. 趋势预测筛选（OR关系）
  if (filters.aiTrendUp || filters.aiTrendDown || filters.aiTrendSideways) {
    if (!trendPrediction) return false;

    const trendMatched =
      (filters.aiTrendUp && trendPrediction.direction === 'up') ||
      (filters.aiTrendDown && trendPrediction.direction === 'down') ||
      (filters.aiTrendSideways && trendPrediction.direction === 'sideways');

    if (!trendMatched) return false;
  }

  // 2. 置信度范围筛选（0-1转换为0-100）
  if (filters.aiConfidenceRange.min !== undefined || filters.aiConfidenceRange.max !== undefined) {
    if (!trendPrediction) return false;

    const confidencePercent = trendPrediction.confidence * 100;

    if (
      filters.aiConfidenceRange.min !== undefined &&
      confidencePercent < filters.aiConfidenceRange.min
    ) {
      return false;
    }
    if (
      filters.aiConfidenceRange.max !== undefined &&
      confidencePercent > filters.aiConfidenceRange.max
    ) {
      return false;
    }
  }

  // 3. 各维度评分范围筛选（综合、技术面、形态、趋势、风险）
  // ... 类似的区间判断逻辑

  // 4. 相似形态匹配要求
  if (filters.aiRequireSimilarPatterns) {
    if (!similarPatterns || similarPatterns.length === 0) {
      return false;
    }
  }

  // 5. 最低相似度筛选（0-1转换为0-100）
  if (filters.aiMinSimilarity !== undefined) {
    if (!similarPatterns || similarPatterns.length === 0) {
      return false;
    }

    const hasMatchedPattern = similarPatterns.some(
      (pattern) => pattern.similarity * 100 >= filters.aiMinSimilarity
    );

    if (!hasMatchedPattern) return false;
  }

  return true;
}
```

### 关键设计决策

#### 1. 数值转换策略

- **置信度**：内部存储 0-1，UI 显示 0-100%
- **相似度**：内部存储 0-1，UI 显示 0-100%
- **评分**：直接使用 0-100，无需转换

**原因**：AI 分析服务返回的是 0-1 的小数，但用户更习惯百分比显示。

#### 2. 筛选逻辑设计

- **趋势预测**：勾选多项为 OR 关系（满足任一即可）
- **评分范围**：同时满足 min 和 max（AND 关系）
- **不同类别**：AND 关系（必须同时满足）

**原因**：符合用户的直觉理解，提供灵活的筛选组合。

#### 3. 无数据处理

没有 AI 分析数据的股票会被自动过滤掉。

**原因**：

- 避免展示不完整的信息
- 确保筛选结果的准确性
- 促使用户先执行“一键分析”

#### 4. 性能优化

- 所有计算在 Web Worker 中进行
- 使用防抖机制（300ms）避免频繁重算
- 只计算激活的筛选项

## 💾 筛选缓存优化

### 优化背景

**问题 1**: 原实现在点击"一键分析"时保存筛选条件，导致需要重新查询 API 获取数据，速度慢。

**问题 2**: 切换路由后再回来，筛选条件恢复为默认值。

- **原因**：自动保存的 useEffect 在组件挂载时立即执行，在恢复逻辑完成前就保存了默认值，覆盖了 localStorage 中的正确值。

**解决方案**:

- 移除"一键分析"时的保存逻辑
- 改为筛选条件变化时自动保存（防抖 300ms）
- **添加恢复标记**，确保在恢复完成前不触发自动保存
- 页面卸载时也保存（兜底机制）

### 工作流程

```
用户操作筛选条件
       │
       ▼
┌──────────────┐
│ 状态更新     │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 防抖300ms    │ ◄── 如果300ms内有新变化，重置计时器
└──────┬───────┘
       │
       ▼ (300ms后无新变化)
┌──────────────┐
│ 保存到       │
│ localStorage │
└─────────────┘
       │
       ▼
┌──────────────┐
│ 纯前端筛选   │ ◄── 不触发API请求，速度快
└──────────────┘
```

### 恢复标记机制

**问题**: 组件挂载时，自动保存的 useEffect 会在恢复逻辑完成前执行，导致默认值覆盖 localStorage 中的正确值。

**解决**: 使用 ref 标记恢复状态

```typescript
// 标记是否已完成初始恢复
const isRestoredRef = useRef(false);

// 恢复逻辑完成后设置标记
useEffect(() => {
  const hydrate = async () => {
    await loadCachedData();
    const prefs = loadOpportunityFilterPrefs();
    if (prefs) {
      applyOpportunityFilterPrefsToState(prefs, actions);
    }
    // 标记恢复完成
    isRestoredRef.current = true;
  };
  void hydrate();
}, [loadCachedData]);

// 自动保存时检查标记
useEffect(
  () => {
    // 如果还未完成初始恢复，不执行保存
    if (!isRestoredRef.current) {
      return;
    }

    const timer = setTimeout(() => {
      saveOpportunityFilterPrefs(prefs);
    }, 300);
    return () => clearTimeout(timer);
  },
  [
    /* 所有依赖 */
  ]
);
```

### 技术细节

1. **防抖机制**: 300ms 防抖避免频繁写入 localStorage
2. **双重保障**: 实时保存 + 卸载保存兜底
3. **依赖管理**: 所有筛选条件加入依赖数组

## 🎯 专业筛选模式

### 三种核心筛选模式

#### 🟢 模式 A：传统门槛模式

- **适用人群**: 习惯传统指标筛选的用户
- **逻辑**: 所有勾选的条件必须同时满足（AND 关系）
- **优点**: 逻辑简单直观，结果确定性高

#### 🔵 模式 B：加权平衡模式（推荐）

- **适用人群**: 希望捕捉潜力股的专业交易者
- **逻辑**: 计算综合得分 = Σ(单项得分 × 权重)，总分达标即可入选
- **建议权重**: 置信度 0.3, 总分 0.4, 技术 0.2, 风险 0.1
- **优点**: 允许"长板效应"，避免误杀优质标的

#### 🟡 模式 C：时效性敏感模式

- **适用人群**: 短线交易者、高频监控者
- **逻辑**: AI 分析结果随时间推移自动打折
- **优点**: 强迫关注最新市场动态

### 高级功能

#### 1. 加权评分配置

- **置信度**: AI 对预测方向的把握程度
- **综合评分**: AI 给出的整体推荐分
- **技术面评分**: 基于 RSI、MACD 等技术指标的强度
- **风险评分**: 分数越高代表风险越低

#### 2. 一致性校验

- **功能**: 自动过滤"技术指标看涨但风险极高"的矛盾信号
- **示例**: 预测"看涨"但风险评分低于 40 分 → 剔除

#### 3. 回测胜率联动

- **功能**: 要求相似形态在历史上的实际盈利比例达到标准
- **建议**: 设置"最低历史胜率"为 60%
