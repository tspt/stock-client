# 智能预警系统增强功能说明

## 功能概述

本次更新为破忒头工具的智能预警系统新增了以下高级预警功能：

### 1. 支撑/阻力位突破预警

**功能描述：**

- 自动识别股票的支撑位和阻力位
- 当价格突破阻力位或跌破支撑位时发出提醒
- 基于局部高低点聚类算法，智能识别关键价位

**技术实现：**

- 使用局部极值检测算法识别高点和低点
- 通过价格水平聚类算法合并相近的支撑/阻力位
- 支持配置回溯周期、聚类容差等参数

**使用场景：**

- 突破交易策略：捕捉价格突破关键阻力位的时机
- 止损设置：监控支撑位是否被跌破
- 技术分析辅助：自动识别重要的技术位

### 2. 成交量异常放大预警

**功能描述：**

- 实时监控成交量变化
- 当成交量超过历史均量的指定倍数时发出提醒
- 可自定义检测周期和倍数阈值

**技术实现：**

- 计算指定周期的历史平均成交量
- 检测当前成交量相对于均量的倍数
- 支持放量（spike）和缩量（shrink）两种异常类型

**使用场景：**

- 捕捉主力异动：大资金进场或离场
- 验证价格突破：放量突破更有说服力
- 发现潜在机会：异常放量往往预示重要行情

### 3. 技术指标金叉/死叉预警

**功能描述：**

- 支持多种技术指标的金叉/死叉检测
- 包括 MACD、KDJ、RSI、均线（MA）等指标
- 可自定义均线周期（仅 MA 指标）

**支持的指标：**

- **MACD**：平滑异同移动平均线，趋势跟踪指标
- **KDJ**：随机指标，超买超卖判断
- **RSI**：相对强弱指标，动量 oscillator
- **MA**：移动平均线，可自定义快慢周期

**技术实现：**

- 实时计算各技术指标的快线和慢线
- 检测快线上穿慢线（金叉）或下穿慢线（死叉）
- 提供详细的指标数值和交叉状态

**使用场景：**

- 趋势确认：金叉确认上涨趋势，死叉确认下跌趋势
- 买卖点提示：结合多个指标提高准确性
- 技术分析自动化：无需手动盯盘

## 使用方法

### 创建新的预警

1. 在股票列表或详情页点击"设置提醒"按钮
2. 选择提醒类型：

   - **价格提醒**：传统的价格涨到/跌到提醒
   - **幅度提醒**：涨跌幅百分比提醒
   - **支撑阻力位**：突破关键价位提醒（新增）
   - **成交量异常**：成交量异动提醒（新增）
   - **指标金叉/死叉**：技术指标交叉提醒（新增）

3. 配置触发条件：

   - 支撑阻力位：选择"突破阻力位"或"跌破支撑位"
   - 成交量异常：设置倍数阈值（如 2.0 倍）和检测周期（如 20 天）
   - 指标交叉：选择指标类型（MACD/KDJ/RSI/MA），选择"金叉"或"死叉"

4. 设置时间周期和通知方式
5. 保存提醒

### 查看预警列表

在"智能预警"页面可以查看所有已设置的提醒，包括：

- 提醒类型（用不同颜色标签区分）
- 触发条件详细说明
- 当前价格和状态
- 时间周期和通知方式

## 技术架构

### 核心模块

1. **类型定义** (`src/types/stock.ts`)

   - 扩展 `AlertType` 枚举，新增三种提醒类型
   - 扩展 `AlertCondition` 枚举，新增突破和金叉/死叉条件
   - 新增 `IndicatorType` 枚举，定义支持的指标类型
   - 在 `PriceAlert` 接口中添加新字段

2. **算法实现** (`src/utils/alertEnhancements.ts`)

   - `identifySupportResistance()`: 支撑阻力位识别
   - `detectVolumeAnomaly()`: 成交量异常检测
   - `detectMACDCross()`: MACD 金叉/死叉检测
   - `detectKDJCross()`: KDJ 金叉/死叉检测
   - `detectRSICross()`: RSI 金叉/死叉检测
   - `detectMACross()`: MA 金叉/死叉检测

3. **状态管理** (`src/stores/alertStore.ts`)

   - 更新 `checkAlerts()` 方法，支持传入 K 线数据
   - 新增三种提醒类型的检查逻辑
   - 实现回退机制防止重复触发

4. **通知服务** (`src/services/alerts/notification.ts`)

   - 扩展通知内容构建逻辑
   - 为新提醒类型生成友好的通知文本

5. **UI 组件**
   - `AlertSettingModal.tsx`: 支持新提醒类型的配置表单
   - `AlertPage.tsx`: 展示新提醒类型的详细信息

### 数据流

```
用户设置提醒 → 保存到 localStorage
     ↓
定时轮询获取行情数据
     ↓
checkAlerts() 检查所有启用的提醒
     ↓
根据提醒类型调用相应的检测算法
     ↓
满足条件 → 触发提醒 → 发送通知
```

## 注意事项

1. **K 线数据依赖**：

   - 支撑阻力位、成交量异常、指标交叉提醒需要 K 线数据
   - 目前仅在有机会分析数据的股票上生效
   - 后续可扩展为自动获取 K 线数据

2. **性能优化**：

   - 支撑阻力位识别使用聚类算法，计算量较大
   - 建议合理设置回溯周期（默认 60 天）
   - 避免同时设置过多的复杂提醒

3. **准确性说明**：

   - 支撑阻力位识别基于历史数据，仅供参考
   - 成交量异常检测受市场整体环境影响
   - 技术指标信号应结合其他分析方法使用

4. **回退机制**：
   - 所有提醒都实现了回退机制
   - 触发后需价格回退到另一侧才能再次触发
   - 避免同一条件下反复提醒

## 未来优化方向

1. **智能参数推荐**：根据股票特性自动推荐合适的参数
2. **多条件组合**：支持多个条件同时满足才触发
3. **机器学习优化**：基于历史数据优化支撑阻力位识别算法
4. **批量设置**：支持为多只股票批量设置相同类型的提醒
5. **提醒统计**：统计各类型提醒的触发频率和准确率
