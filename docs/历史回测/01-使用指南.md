# 历史回测 - 使用指南

> 📚 **相关文档**: [技术实现](./02-技术实现.md) | [返回上级](./README.md)

本文档面向**普通用户**，介绍历史回测功能的使用方法和结果解读。

---

## 🎯 功能概览

历史回测功能基于本地 IndexedDB 存储的全量历史 K 线数据，对交易信号进行回测验证，帮助用户评估策略有效性并优化参数配置。

### 核心能力

- ✅ **全量回测** - 基于本地历史数据，无需联网即可执行回测
- ✅ **并行计算** - Web Worker 多线程处理，大幅提升回测速度
- ✅ **ML 模型集成** - 集成机器学习买点模型，智能识别交易信号
- ✅ **灵活筛选** - 支持市场类型、股票名称、时间范围等多维度筛选
- ✅ **结果持久化** - 回测结果自动保存，下次打开页面自动加载
- ✅ **数据导出** - 支持单股票和批量导出为 JSON 格式

## ⚙️ 回测流程

### 步骤 1：准备历史数据

在执行回测之前，需要确保 IndexedDB 中有足够的历史数据：

1. **进入机会分析页面**
2. **触发数据同步**（等待数据加载完成）
3. **确认数据存储**（可在数据管理页面查看）

**注意**：如果没有历史数据，回测将无法执行。

### 步骤 2：配置回测参数

点击"筛选"按钮打开高级筛选面板，配置以下参数：

#### 基础筛选

| 参数     | 说明           | 可选值                       |
| -------- | -------------- | ---------------------------- |
| 市场类型 | 选择股票市场   | 沪深主板 / 创业板 / 科创板等 |
| 名称类型 | 过滤特殊股票   | 非 ST / 包含 ST / 全部       |
| 时间范围 | 回测的时间跨度 | 近 30 天 / 60 天 / 90 天等   |

#### 高级筛选

| 参数     | 说明                | 默认值 | 建议             |
| -------- | ------------------- | ------ | ---------------- |
| 最低胜率 | 近 3 日胜率最低阈值 | 0%     | 根据风险偏好调整 |

**使用建议**：

- 首次回测建议使用默认配置
- 熟悉后可根据需求调整筛选条件
- 筛选条件越严格，回测速度越快

### 步骤 3：执行回测

1. **点击"开始回测"按钮**
2. **等待数据加载**（从 IndexedDB 读取历史数据）
3. **观察进度条**（显示回测进度）
4. **等待完成**（回测完成后自动展示结果）

**耗时参考**：

- 100 只股票：约 10-30 秒
- 500 只股票：约 1-3 分钟
- 1000+ 只股票：约 3-10 分钟

**注意**：回测过程中请勿关闭页面。

### 步骤 4：查看回测结果

回测完成后，结果会自动展示：

#### 左侧：股票列表

- 显示所有有信号的股票
- 按信号数量排序
- 点击股票查看详细信息

#### 右侧：信号详情

- 显示选中股票的所有信号
- 包含信号日期、入场价格、收益率等信息
- 支持导出数据

## 📊 结果解读

### 关键指标

| 指标       | 含义               | 解读要点              |
| ---------- | ------------------ | --------------------- |
| 信号数量   | 触发的交易信号总数 | 越多表示策略越活跃    |
| 平均收益率 | 所有信号的平均收益 | >0% 表示策略整体盈利  |
| 胜率       | 盈利信号占比       | >50% 表示策略较为可靠 |
| 最大收益   | 单笔最高收益       | 反映策略的爆发力      |
| 最大亏损   | 单笔最大亏损       | 反映策略的风险控制    |

### 信号分类

**成功信号**（收益率 > 0%）：

- 🟢 绿色标记
- 表示买入后价格上涨
- 理想情况下应占多数

**失败信号**（收益率 ≤ 0%）：

- 🔴 红色标记
- 表示买入后价格下跌或持平
- 需要分析失败原因

## 💾 数据导出

### 导出单个股票

1. **在左侧列表中选择股票**
2. **点击"导出"按钮**
3. **输入买点日期**（每行一个日期，8 位数字格式）
4. **点击"确认导出"**
5. **等待文件下载**

**文件格式**：JSON  
**文件名**：`backtest_{股票代码}_{日期}.json`  
**包含内容**：

- 股票基本信息
- K 线数据
- 信号列表
- 买点验证结果

### 批量导出

1. **点击"批量导出"按钮**
2. **选择要导出的股票**（可多选）
3. **点击"确认导出"**
4. **等待所有文件生成**

**注意**：

- 批量导出会生成多个文件
- 建议在网速较好时操作
- 大数量导出可能需要较长时间

## ❓ 常见问题

### Q1: 为什么回测很慢？

**A**: 可能的原因：

1. **数据量大**：股票数量多或时间跨度长
2. **电脑性能**：CPU 性能影响计算速度
3. **浏览器限制**：某些浏览器对 Web Worker 有限制

**解决方案**：

- 减少筛选的股票数量
- 缩短时间范围
- 关闭其他占用资源的程序

### Q2: 回测结果为空怎么办？

**A**: 检查以下几点：

1. **是否有历史数据**：在数据管理页面确认
2. **筛选条件是否过严**：尝试放宽筛选条件
3. **时间范围是否合适**：确保时间范围内有数据

**解决方案**：

- 先在机会分析页面触发数据同步
- 使用默认筛选条件重新回测
- 检查控制台错误日志

### Q3: 如何验证回测结果的准确性？

**A**: 可以通过以下方式验证：

1. **手动抽查**：随机选择几只股票，手动核对 K 线数据
2. **对比实盘**：将回测结果与实盘表现对比
3. **交叉验证**：使用不同的时间范围回测，观察一致性

### Q4: 回测结果可以保存多久？

**A**:

- 回测结果永久保存在 IndexedDB 中
- 除非手动清空或清除浏览器数据
- 建议定期导出重要结果作为备份

### Q5: 如何提高回测速度？

**A**: 优化建议：

1. **减少股票数量**：使用筛选条件缩小范围
2. **缩短时间范围**：只回测最近的数据
3. **关闭其他应用**：释放系统资源
4. **使用高性能浏览器**：Chrome 或 Edge 性能较好

### Q6: ML 模型是如何工作的？

**A**:

- ML 模型基于历史数据训练
- 自动识别 K 线形态和技术指标
- 输出买点的置信度评分
- 详细原理见 [02-技术实现](./02-技术实现.md)
