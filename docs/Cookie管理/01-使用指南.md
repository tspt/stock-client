# Cookie 管理使用指南

## 概述

Cookie 管理页面提供了智能化的东方财富 API Cookie 池化管理功能，帮助您：

- 🔄 **自动轮换** - 多个 Cookie 轮流使用，避免单个 Cookie 频繁请求被封
- 📊 **健康监控** - 实时跟踪每个 Cookie 的使用情况和健康状态
- 🤖 **智能选择** - 基于健康评分的加权随机算法，优先使用高质量 Cookie
- 💾 **数据备份** - 支持导出/导入 Cookie，方便备份和迁移

---

## 🎯 核心概念

### 什么是 Cookie 池？

Cookie 池是一个存储多个东方财富 API Cookie 的容器。系统会：

1. **智能选择** - 从池中选择一个健康的 Cookie 用于 API 请求
2. **自动轮换** - 请求失败时自动切换到其他 Cookie
3. **健康管理** - 跟踪每个 Cookie 的成功/失败次数，计算健康评分

### 为什么需要 Cookie 池？

东方财富 API 有严格的反爬机制：

- ❌ 单个 Cookie 频繁请求会被封禁
- ❌ Cookie 会过期失效（几小时到几天）
- ✅ 多个 Cookie 轮换可以大幅降低被封风险
- ✅ 自动切换确保持续可用

### 健康评分是什么？

健康评分（0-100 分）反映 Cookie 的质量：

- **高分（80-100）** - 成功率高，推荐使用
- **中分（50-79）** - 一般可用，备选方案
- **低分（0-50）** - 失败率高，建议替换

评分根据以下因素计算：

- 成功率（成功次数 / 总请求次数）
- 使用频率（最近是否经常使用）
- 年龄（创建时间长短）

## 🚀 快速开始

### 1. 访问 Cookie 管理页面

在应用左侧导航栏点击 **"Cookie 管理"** 进入页面。

### 2. 查看当前状态

页面顶部显示统计信息：

```
┌─────────────────────────────────────────┐
│ 总数量: 5    活跃: 4    平均健康: 85.2  │
│ 总请求: 1,234    成功率: 96.5%          │
└─────────────────────────────────────────┘
```

- **总数量** - Cookie 池中 Cookie 总数
- **活跃** - 当前可用的 Cookie 数量
- **平均健康** - 所有活跃 Cookie 的平均健康评分
- **总请求** - 累计 API 请求次数
- **成功率** - 成功请求占比

### 3. 添加第一个 Cookie

#### 方法 A：手动添加（推荐新手）

1. 点击 **"➕ 添加 Cookie"** 按钮
2. 打开浏览器访问 https://data.eastmoney.com
3. 按 F12 打开开发者工具
4. 切换到 **Network** 标签
5. 刷新页面
6. 找到任意 eastmoney.com 的请求
7. 复制 **Request Headers** 中的 **Cookie** 值
8. 粘贴到文本框中
9. 点击 **"确定"**

#### 方法 B：自动获取（推荐高级用户）

1. 点击 **"🤖 自动获取"** 按钮
2. 设置获取数量（默认 50 个）
3. 点击 **"开始获取"**
4. 等待自动完成（约 1-2 分钟）
5. 系统会自动去重并添加到池中

**注意**：自动获取会打开隐藏的浏览器窗口，请勿关闭应用。

## 📋 功能详解

### Cookie 列表

表格显示所有 Cookie 的详细信息：

| 列名      | 说明                       |
| --------- | -------------------------- |
| ID        | Cookie 唯一标识（前 8 位） |
| 健康评分  | 0-100 分，颜色表示状态     |
| 状态      | 活跃/停用                  |
| 成功/失败 | 累计成功和失败次数         |
| 最后使用  | 上次使用时间               |
| 创建时间  | Cookie 添加时间            |
| 操作      | 删除、测试等操作           |

#### 健康评分颜色

- 🟢 **绿色（≥80）** - 优秀
- 🟡 **黄色（50-79）** - 良好
- 🔴 **红色（<50）** - 较差

### 添加 Cookie

#### 手动添加步骤

1. **获取 Cookie**

   ```
   浏览器 → data.eastmoney.com → F12 → Network → 刷新
   → 选择请求 → Request Headers → 复制 Cookie
   ```

2. **粘贴到应用**

   - 点击 "➕ 添加 Cookie"
   - 粘贴 Cookie 字符串
   - 点击 "确定"

3. **验证结果**
   - 系统会自动验证格式
   - 检查是否重复
   - 添加到池中

#### 批量添加

可以一次性粘贴多个 Cookie（每行一个）：

```
cookie1=value1; cookie2=value2; ...
cookie1=value3; cookie2=value4; ...
```

### 删除 Cookie

#### 单个删除

1. 找到要删除的 Cookie
2. 点击操作列的 **"🗑️"** 图标
3. 确认删除

#### 批量清理

1. 点击 **"🧹 清理失效 Cookie"** 按钮
2. 系统会自动删除：
   - 健康评分 < 20 的 Cookie
   - 连续失败 > 10 次的 Cookie
   - 超过 7 天未使用的 Cookie

### 测试 Cookie

#### 单个测试

1. 找到要测试的 Cookie
2. 点击操作列的 **"🔍"** 图标
3. 系统会发送测试请求
4. 查看测试结果（成功/失败）

#### 批量测试

1. 点击 **"🔄 测试所有 Cookie"** 按钮
2. 系统会依次测试所有 Cookie
3. 更新健康评分和状态

### 导出数据

#### 导出为文本

1. 点击 **"📥 导出"** 按钮
2. 选择导出范围：
   - **全部** - 导出所有 Cookie
   - **仅活跃** - 只导出活跃的 Cookie
   - **仅失效** - 只导出失效的 Cookie
3. 点击 **"确定"**
4. 文件会自动下载为 `cookies_backup_YYYY-MM-DD.txt`

#### 导出格式

```text
# Cookie 备份文件
# 导出时间: 2026-04-23 15:30:00
# 总数: 5

cookie1=value1; cookie2=value2; ...
cookie1=value3; cookie2=value4; ...
...
```

### 导入数据

#### 从文本导入

1. 点击 **"📤 导入"** 按钮
2. 选择之前导出的 `.txt` 文件
3. 系统会自动解析并添加
4. 查看导入结果（成功/跳过/失败）

#### 注意事项

- 重复的 Cookie 会自动跳过
- 格式错误的 Cookie 会显示错误信息
- 导入不会覆盖现有 Cookie

## ⚙️ 高级功能

### 自动获取 Cookie

#### 工作原理

1. 打开隐藏的 BrowserWindow
2. 访问 data.eastmoney.com
3. 拦截响应中的 Set-Cookie 头
4. 提取并添加到池中
5. 关闭浏览器窗口
6. 重复上述步骤 N 次

#### 使用建议

- **获取数量** - 建议 30-50 个
- **获取频率** - 每天 1-2 次即可
- **最佳时机** - 应用启动时或 Cookie 数量低于 5 个时

#### 注意事项

⚠️ **首次获取较慢** - 需要加载浏览器，约 10-15 秒  
⚠️ **不要关闭窗口** - 获取过程中请勿关闭应用  
⚠️ **网络要求** - 需要稳定的网络连接

### 智能选择策略

系统使用**加权随机算法**选择 Cookie：

1. 过滤出所有活跃的 Cookie
2. 按健康评分降序排序
3. 取前 5 个高质量 Cookie
4. 根据健康评分加权随机选择

**优势**：

- ✅ 避免总是使用同一个 Cookie
- ✅ 优先使用高质量 Cookie
- ✅ 自动负载均衡

### 健康评分算法

```
健康评分 = 基础分 × 成功率 × 年龄衰减因子

其中：
- 基础分 = 100
- 成功率 = 成功次数 / 总请求次数
- 年龄衰减 = 超过24小时后每小时减1分
```

**示例**：

- Cookie A: 100 次请求，95 次成功，创建 2 小时
  - 评分 = 100 × 0.95 × 1.0 = 95 分 ✅
- Cookie B: 50 次请求，30 次成功，创建 48 小时
  - 评分 = 100 × 0.60 × 0.76 = 45.6 分 ⚠️

## ❓ 常见问题

### Q1: 为什么我的 Cookie 很快就失效了？

**A**: 可能原因：

1. Cookie 本身有效期短（几小时）
2. 请求频率过高被临时封禁
3. IP 地址被限制

**解决方案**：

- 增加 Cookie 数量（建议保持 5-10 个）
- 降低请求频率
- 使用自动获取定期更新

### Q2: 如何知道哪个 Cookie 正在使用？

**A**: 系统采用随机选择策略，不会固定使用某个 Cookie。您可以在操作日志中查看最近的请求记录。

### Q3: Cookie 池为空怎么办？

**A**:

1. 立即添加新的 Cookie（手动或自动）
2. 系统会在 Cookie 池为空时使用环境变量中的备用 Cookie
3. 建议设置提醒，当 Cookie 数量低于 3 个时收到通知

### Q4: 导出的 Cookie 安全吗？

**A**:

- Cookie 存储在本地 IndexedDB 中，不会上传到服务器
- 导出的文件保存在您的电脑上
- 建议妥善保管导出文件，不要分享给他人
- 可选：启用加密存储（需在代码中配置）

### Q5: 自动获取失败怎么办？

**A**: 可能原因：

1. 网络连接问题
2. 东方财富网站临时不可用
3. 浏览器窗口被阻止

**解决方案**：

- 检查网络连接
- 稍后重试
- 改用手动添加方式

### Q6: 多久需要更新一次 Cookie？

**A**: 建议：

- **日常使用** - 每周自动获取一次
- **高频使用** - 每天自动获取一次
- **低频使用** - 每月检查一次即可

关键指标：当活跃 Cookie 数量 < 3 时，应立即补充。

## 💡 最佳实践

### 1. 保持足够的 Cookie 数量

- **最低要求** - 至少 3 个活跃 Cookie
- **推荐数量** - 5-10 个活跃 Cookie
- **理想状态** - 10+ 个活跃 Cookie

### 2. 定期监控健康状态

- 每天查看一次统计面板
- 关注成功率变化趋势
- 及时删除低分 Cookie

### 3. 建立备份习惯

- 每周导出一次 Cookie 备份
- 保存到安全的位置
- 重要时期（如交易高峰期）增加备份频率

### 4. 合理使用自动获取

- 设置在应用启动时自动获取
- 避免短时间内多次获取
- 获取后验证新增 Cookie 质量

### 5. 注意请求频率

- 避免短时间内大量请求
- 实现请求间隔控制
- 监控 API 返回的状态码
