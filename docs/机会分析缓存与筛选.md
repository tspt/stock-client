# 机会分析：缓存、纯前端筛选与偏好记忆

## 1. 服务器返回的数据能否只做纯前端筛选？

**可以。** 一键分析完成后，每条股票具备：

- **基础行情与指标**：现价、市值、换手率、市盈率、KDJ、各周期 MA 等（来自当次接口与本地计算）。
- **K 线序列**：按所选周期与根数拉取后存入 `klineDataCache`（IndexedDB 与内存 Map）。

在此基础上，下列能力**不依赖再次请求接口**，仅在页面内用缓存重算或过滤即可（**实现上**：见下文「Web Worker 筛选管线」）。

| 能力 | 数据来源 | 说明 |
|------|----------|------|
| 数据筛选（轻量） | `StockOpportunityData` 字段 | 价格、市值、换手率、市盈率、KDJ-J 等在主线程先过滤 |
| 数据筛选（依赖 K 线） | `klineDataCache` + Worker 内计算 | 涨跌停统计等需逐根 K 线的项在 Worker 中重算 |
| 横盘 | `klineDataCache` + `calculateConsolidationInLookback` | 按用户「检索根数 / 连续根数 / 阈值 / MA10」在 Worker 中重算 |
| 趋势线 | `klineDataCache` + `calculateTrendLineInLookback` | 按用户 M、N 在 Worker 中重算 |
| 急跌/急涨 | `volumeSurgePatterns` + 缓存 K 线 | 周期与横盘阈值变化时在 Worker 中重算关联横盘段 |

**限制**：若修改 **K 线周期（日/周/月/年）** 或 **K 线根数**，与已缓存的 K 线不一致，需要重新一键分析拉取新数据。头部「市场 / 名称类型」决定参与分析的股票集合，变更后同样需要重新分析。

---

## 2. 本地存储分工

| 存储 | 键/位置 | 内容 |
|------|---------|------|
| **IndexedDB** | `opportunity` 库 `latest` | 上次分析结果列表、`period`、`count`、**K 线缓存** `klineDataCache` |
| **localStorage** | `opportunity_filter_prefs` | 上次点击「一键分析」时的**查询条**（市场、名称类型、周期、K 线数量）与**全部筛选表单**状态 |

---

## 3. 下次打开页面时的行为

1. 从 IndexedDB 恢复分析数据与 K 线缓存（若存在）。
2. 从 localStorage 读取筛选偏好并填充表单。
3. **若已有缓存数据**：头部「周期」「K 线数量」**以缓存为准**（与数据一致），其余筛选项仍用偏好还原，列表按当前筛选条件对缓存数据做纯前端过滤。
4. **若无缓存数据**：周期与 K 线数量也按偏好还原。

---

## 4. Web Worker 筛选管线

机会分析页的「全量筛选」在 **`useOpportunityFilterEngine`** 中协调，避免主线程长时间阻塞：

1. **主线程**：对 `analysisData` 先做 **轻量过滤**（`passLightFilters`：价格、市值、换手率、市盈率、KDJ-J 等仅依赖 `StockOpportunityData` 的字段）。
2. **条数上限**：轻量过滤后的列表若超过 `MAX_OPPORTUNITY_KLINE_CACHE_ENTRIES`，会 **优先保留 K 线缓存命中的标的**，再按原顺序补足，再交给 Worker（与 IndexedDB 中 K 线缓存条数策略一致）。
3. **Worker**（`opportunityFilterWorker.ts`）：持有 K 线 Map 的副本；横盘、趋势线、急跌急涨模式、涨跌停统计等依赖 K 线或重算成本较高的步骤在此执行；筛选条件变化时 **300ms 防抖** 后发起新任务，旧任务通过 `requestId` 与 `cancel` 丢弃，避免结果乱序回写。
4. **K 线同步**：`klineDataCache` 变更时，向 Worker 发送 **`set-data-full`**（变化比例大时）或 **`set-data-patch`**（增量 upsert / remove），减少重复传输。
5. **状态与提示**：筛选进行中 `filtering` 为 true；因条数上限未参与本次 Worker 筛选的标的会进入 `skipped` 列表供界面提示。

---

## 5. 按钮行为

- **一键分析**：在发起请求前将当前查询条件与筛选表单写入 `opportunity_filter_prefs`；分析结束后结果与 K 线仍写入 IndexedDB（沿用原有逻辑）。
- **重置查询**（「一键分析」右侧）：仅恢复顶部 **市场、名称类型、周期、K 线数量**；若本地已有 `opportunity_filter_prefs`，只覆盖其中对应字段。
- **重置筛选**：恢复 **数据筛选 / 横盘 / 趋势线 / 急跌急涨** 等下方表单默认值（不动顶部查询条）；若已有偏好文件，只覆盖筛选相关字段。
