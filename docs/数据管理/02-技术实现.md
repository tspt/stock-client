## 💾 数据存储策略

### 存储方案对比

| 数据类型     | 存储方式     | 原因                         |
| ------------ | ------------ | ---------------------------- |
| 股票列表     | localStorage | 数据量小（~500KB），结构简单 |
| 板块基础信息 | localStorage | 数据量小（~50KB），读取频繁  |
| 成分股数据   | IndexedDB    | 数据量大（5-20MB），结构复杂 |
| Cookie       | IndexedDB    | 需要事务支持和索引查询       |

### IndexedDB 实现

#### 数据库结构

```typescript
// src/utils/storage/sectorStocksIndexedDB.ts

const DB_NAME = 'SectorStocksDB';
const DB_VERSION = 1;

// 行业板块表
const INDUSTRY_STORE = 'industrySectors';
// 概念板块表
const CONCEPT_STORE = 'conceptSectors';

interface SectorStockEntry {
  sectorCode: string; // 板块代码（主键）
  sectorName: string; // 板块名称
  stockCount: number; // 成分股数量
  stocks: string[]; // 股票代码数组
  savedAt: number; // 保存时间戳
}
```

#### 核心操作

```typescript
// 获取所有行业板块
export async function getIndustrySectors(): Promise<SectorStockEntry[]> {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open(DB_NAME, DB_VERSION);
    request.onsuccess = (event) => {
      const db = (event.target as IDBOpenDBRequest).result;
      const transaction = db.transaction([INDUSTRY_STORE], 'readonly');
      const store = transaction.objectStore(INDUSTRY_STORE);
      const getAllRequest = store.getAll();

      getAllRequest.onsuccess = () => {
        resolve(getAllRequest.result);
      };
    };
  });
}

// 清空所有数据
export async function clearSectorStocksDB(): Promise<void> {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open(DB_NAME, DB_VERSION);
    request.onsuccess = (event) => {
      const db = (event.target as IDBOpenDBRequest).result;
      const transaction = db.transaction([INDUSTRY_STORE, CONCEPT_STORE], 'readwrite');

      transaction.objectStore(INDUSTRY_STORE).clear();
      transaction.objectStore(CONCEPT_STORE).clear();

      transaction.oncomplete = () => resolve();
      transaction.onerror = () => reject(transaction.error);
    };
  });
}
```

## 📤 数据导入导出

### 导出功能

```typescript
// src/utils/export/sectorStocksExport.ts
export async function exportSectorStocksToJSON(): Promise<void> {
  try {
    // 1. 从 IndexedDB 读取所有数据
    const [industrySectors, conceptSectors] = await Promise.all([
      getIndustrySectors(),
      getConceptSectors(),
    ]);

    // 2. 构建导出数据结构
    const exportData = {
      version: '1.0',
      exportDate: new Date().toISOString(),
      industrySectors,
      conceptSectors,
      statistics: {
        industryCount: industrySectors.length,
        conceptCount: conceptSectors.length,
      },
    };

    // 3. 转换为 JSON 字符串
    const jsonContent = JSON.stringify(exportData, null, 2);

    // 4. 创建 Blob 并触发下载
    const blob = new Blob([jsonContent], { type: 'application/json' });
    const url = URL.createObjectURL(blob);

    const a = document.createElement('a');
    a.href = url;
    a.download = `sector_stocks_${formatTimestamp(Date.now())}.json`;
    document.body.appendChild(a);
    a.click();
    document.body.removeChild(a);
    URL.revokeObjectURL(url);

    logger.info('成分股数据导出成功');
  } catch (error) {
    logger.error('导出成分股数据失败:', error);
    throw error;
  }
}
```

### 导入功能

```typescript
export async function importSectorStocksFromJSON(
  file: File,
  mode: 'overwrite' | 'merge' = 'overwrite'
): Promise<{ success: boolean; message: string }> {
  try {
    // 1. 验证文件格式
    if (!file.name.endsWith('.json')) {
      return { success: false, message: '只支持 JSON 格式文件' };
    }

    // 2. 读取文件内容
    const text = await file.text();
    let data: any;

    try {
      data = JSON.parse(text);
    } catch {
      return { success: false, message: 'JSON 格式错误，文件可能已损坏' };
    }

    // 3. 验证数据结构
    if (!data.industrySectors || !data.conceptSectors) {
      return { success: false, message: '无效的数据文件格式' };
    }

    // 4. 根据模式导入数据
    if (mode === 'overwrite') {
      // 清空现有数据
      await clearSectorStocksDB();
    }

    // 5. 写入 IndexedDB
    await saveSectorsToDB(data.industrySectors, 'industry');
    await saveSectorsToDB(data.conceptSectors, 'concept');

    const totalCount = data.industrySectors.length + data.conceptSectors.length;
    return {
      success: true,
      message: `成功导入 ${totalCount} 个板块的成分股数据`,
    };
  } catch (error) {
    logger.error('导入成分股数据失败:', error);
    return {
      success: false,
      message: `导入失败: ${error instanceof Error ? error.message : String(error)}`,
    };
  }
}
```

### 缓存状态检查

```typescript
const loadStockListStatus = async () => {
  try {
    const cache = getStorage<{ savedAt: number; stocks: StockInfo[] } | null>(
      CACHE_KEYS.BIYING_STOCK_LIST,
      null
    );

    if (cache && cache.stocks) {
      const now = Date.now();
      const age = now - cache.savedAt;
      const ttl = CACHE_TTL.STOCK_LIST;
      const isExpired = age > ttl;

      setStockListStatus({
        count: cache.stocks.length,
        lastUpdate: cache.savedAt,
        isExpired,
      });
    } else {
      setStockListStatus({ count: 0, isExpired: true });
    }
  } catch (error) {
    logger.error('加载股票列表状态失败:', error);
  }
};
```

## 🐛 常见问题排查

### 问题 1: 数据刷新失败

**症状**：点击刷新按钮后显示错误提示

**排查步骤**：

1. 打开浏览器控制台（F12）
2. 查看 Network 标签，检查 API 请求状态
3. 查看 Console 标签，查找错误日志
4. 检查网络连接是否正常
5. 检查 Cookie 是否有效（Cookie 管理页面）

**常见原因**：

- 网络超时
- Cookie 失效
- API 限流
- 服务器异常

### 问题 2: 导入数据失败

**症状**：选择文件后提示"导入失败"

**排查步骤**：

1. 检查文件格式是否为 `.json`
2. 用文本编辑器打开文件，验证 JSON 格式
3. 检查文件版本是否兼容
4. 查看控制台错误日志

**常见原因**：

- 文件格式错误
- 文件损坏
- 版本不兼容
- 磁盘空间不足

### 问题 3: IndexedDB 数据丢失

**症状**：重启应用后成分股数据消失

**排查步骤**：

1. 检查浏览器是否清除了网站数据
2. 检查磁盘空间是否充足
3. 查看 IndexedDB 是否存在（DevTools → Application → IndexedDB）
4. 检查是否有多个浏览器配置文件

**常见原因**：

- 浏览器清理缓存
- 磁盘空间不足
- 使用了无痕模式
- 数据库版本冲突
