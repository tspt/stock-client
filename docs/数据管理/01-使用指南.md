# 数据管理 - 使用指南

本文档面向**普通用户**，介绍数据管理功能的使用方法和操作说明。

---

## 🎯 功能概览

数据管理页面是破忒头工具的数据中心，统一管理应用中的所有缓存数据，包括股票列表、板块信息和成分股数据。

### 核心能力

- ✅ **股票列表管理** - 查看、刷新、清除 A 股全量数据
- ✅ **板块基础信息** - 管理行业和概念板块元数据
- ✅ **成分股数据** - 导出/导入板块成分股（IndexedDB）
- ✅ **缓存状态监控** - 实时显示数据量和更新状态
- ✅ **一键操作** - 快速刷新、清除、备份数据

---

## 📋 功能详解

### 1️⃣ 股票列表管理

#### 功能说明

股票列表包含 A 股市场所有可交易股票的基础信息（代码、名称、价格等），是其他功能的数据基础。

#### 界面信息

| 指标     | 说明                        |
| -------- | --------------------------- |
| 股票总数 | 当前缓存的股票数量          |
| 最后刷新 | 上次获取数据的时间          |
| 缓存状态 | 有效（绿色）/已过期（红色） |

#### 操作按钮

**强制刷新**

- **作用**：从服务器重新获取最新股票列表
- **使用场景**：
  - 新上市股票未显示
  - 股票信息需要更新
  - 缓存已过期
- **注意事项**：
  - 需要联网
  - 耗时约 3-5 秒
  - 会覆盖现有数据

**清除缓存**

- **作用**：删除本地缓存的股票列表
- **使用场景**：
  - 释放存储空间
  - 解决数据显示异常
- **注意事项**：
  - 清除后需重新刷新才能使用
  - 不影响其他数据

---

### 2️⃣ 板块基础信息管理

#### 功能说明

板块基础信息包含行业板块和概念板块的元数据（板块代码、名称、成分股数量等）。

#### 界面信息

分为两个子卡片：

- **行业板块**：证监会行业分类（如：银行、医药、电子等）
- **概念板块**：市场热点概念（如：人工智能、新能源汽车等）

每个卡片显示：

- 板块数量
- 最后更新时间

#### 操作按钮

**刷新**

- **作用**：从服务器获取最新板块列表
- **使用场景**：
  - 新增板块未显示
  - 板块名称需要更新
- **注意事项**：
  - 行业板块和概念板块需分别刷新
  - 耗时约 1-2 秒

**清除**

- **作用**：删除本地缓存的板块基础信息
- **使用场景**：
  - 板块数据异常
  - 需要重新获取
- **注意事项**：
  - 清除后需重新刷新
  - 不影响成分股数据

---

### 3️⃣ 成分股数据管理（IndexedDB）

#### 功能说明

成分股数据包含每个板块下的具体股票列表，存储在 IndexedDB 中，支持大容量数据存储。

#### 界面信息

分为两个子卡片：

- **行业成分股**：各行业板块的成分股数据
- **概念成分股**：各概念板块的成分股数据

每个卡片显示：

- 板块数量（有多少个板块的成分股已缓存）
- 最后更新时间

#### 操作按钮

**导出数据**

- **作用**：将所有成分股数据导出为 JSON 文件
- **使用场景**：
  - 数据备份
  - 迁移到其他设备
  - 分享给他人
- **操作步骤**：
  1. 点击"导出数据"按钮
  2. 选择保存位置
  3. 等待下载完成
- **文件格式**：JSON
- **文件名**：`sector_stocks_YYYY-MM-DD_HH-mm-ss.json`
- **注意事项**：
  - 文件大小取决于数据量（通常几 MB）
  - 包含完整元数据（板块信息、股票代码、更新时间等）

**导入数据**

- **作用**：从 JSON 文件恢复成分股数据
- **使用场景**：
  - 从备份恢复数据
  - 接收他人分享的数据
  - 更换设备后迁移数据
- **操作步骤**：
  1. 点击"导入数据"按钮
  2. 选择之前导出的 JSON 文件
  3. 等待导入完成
- **导入模式**：覆盖模式（替换现有数据）
- **注意事项**：
  - 仅支持 `.json` 格式文件
  - 必须是本系统导出的文件
  - 导入会覆盖现有数据
  - 大文件可能需要几秒到几十秒

**清空全部**

- **作用**：删除所有成分股数据
- **使用场景**：
  - 释放存储空间
  - 重新开始获取数据
  - 解决数据异常
- **注意事项**：
  - ⚠️ **此操作不可恢复**
  - 会弹出确认对话框
  - 清空后需重新获取成分股

## 💡 使用建议

### 日常维护

1. **定期检查缓存状态**

   - 进入数据管理页面
   - 查看各模块的"缓存状态"
   - 如果显示"已过期"，考虑刷新

2. **重要数据定期备份**

   - 每月导出一次成分股数据
   - 保存到云盘或外部存储
   - 标注日期便于识别

3. **遇到问题先清除缓存**
   - 数据显示异常 → 清除对应缓存后刷新
   - 性能变慢 → 清除不需要的数据
   - 空间不足 → 清空成分股后重新获取

### 最佳实践

#### 新用户初始化流程

1. **首次使用**

   ```
   步骤 1: 刷新股票列表（获取全量股票）
   步骤 2: 刷新行业板块基础信息
   步骤 3: 刷新概念板块基础信息
   步骤 4: 在"成分股大全"页面获取成分股
   步骤 5: 导出成分股数据作为备份
   ```

2. **日常使用**

   ```
   - 股票列表：每周刷新一次（或发现缺失时）
   - 板块信息：每月刷新一次（变化较少）
   - 成分股数据：按需获取，定期备份
   ```

3. **数据迁移**

   ```
   旧设备:
   1. 导出成分股数据
   2. 记录 Cookie（Cookie 管理页面导出）

   新设备:
   1. 安装应用
   2. 导入成分股数据
   3. 导入 Cookie
   4. 刷新股票列表和板块信息
   ```

## ❓ 常见问题

### Q1: 为什么股票列表显示"已过期"？

**A**: 股票列表有 TTL（生存时间），默认 24 小时。过期后可以继续使用，但建议刷新以获取最新数据（新股上市、停牌等）。

### Q2: 成分股数据和板块基础信息有什么区别？

**A**:

- **板块基础信息**：板块本身的元数据（代码、名称、成分股数量），数据量小，存储在 localStorage
- **成分股数据**：每个板块下的具体股票列表，数据量大，存储在 IndexedDB

### Q3: 导入数据失败怎么办？

**A**: 检查以下几点：

1. 文件格式是否为 `.json`
2. 文件是否由本系统导出
3. 文件是否损坏（可用文本编辑器打开检查）
4. 磁盘空间是否充足

### Q4: 清空数据后如何恢复？

**A**:

- 如果有备份：使用"导入数据"功能
- 如果没有备份：
  - 股票列表：点击"强制刷新"
  - 板块信息：点击"刷新"
  - 成分股：在"成分股大全"页面重新获取

### Q5: 数据占用多少存储空间？

**A**: 大致估算：

- 股票列表：~500 KB（5000+ 股票）
- 板块基础信息：~50 KB（200+ 板块）
- 成分股数据：~5-20 MB（取决于板块数量）
- **总计**：通常 < 30 MB

### Q6: 可以只导出部分板块的成分股吗？

**A**: 目前不支持。导出功能会导出所有板块的成分股数据。如果只需要部分数据，可以考虑：

1. 导出全部数据
2. 用文本编辑器手动筛选（不推荐）
3. 等待后续版本支持选择性导出
