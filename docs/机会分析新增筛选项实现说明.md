# 机会分析新增筛选项说明文档

## 概述

本次实现为机会分析页面添加了完整的技术指标与形态筛选功能，包括 RSI、MACD、布林带、K 线形态和趋势形态等高级筛选条件。

所有新增筛选项位于右侧抽屉的「技术指标与形态」面板中，与现有的横盘、趋势线、单日异动等筛选条件并列，共同构成机会分析的完整筛选体系。

---

## 实现的功能

### 1. 技术指标筛选

#### RSI（相对强弱指数）

**含义**：衡量价格变动速度和幅度的动量振荡器，用于判断超买超卖状态。

**参数**：

- **RSI 范围**：支持设置最小值和最大值（0-100）
- **计算周期**：固定 14 周期（标准配置）

**计算公式**：

```
1. 计算价格变化序列
2. 分别计算上涨日和下跌日的平均幅度
3. RS = 平均上涨幅度 / 平均下跌幅度
4. RSI = 100 - (100 / (1 + RS))
```

**建议值**：

- **超卖区域**：RSI < 30（可能反弹）
- **中性区域**：30 ≤ RSI ≤ 70
- **超买区域**：RSI > 70（可能回调）

**数据要求**：至少需要 15 根 K 线

**实现位置**：`src/utils/technicalIndicators.ts` 中的 `calculateRSI`

---

#### MACD（平滑异同移动平均线）

**含义**：通过快慢均线的聚合与分离来判断买卖时机和趋势强度。

**参数**：

- **金叉**：DIF 从下向上穿越 DEA
- **死叉**：DIF 从上向下穿越 DEA
- **背离**：价格创新高/新低而 DIF 未同步

**计算公式**：

```
1. EMA(12) = 12日指数移动平均
2. EMA(26) = 26日指数移动平均
3. DIF = EMA(12) - EMA(26)
4. DEA = DIF的9日EMA
5. MACD柱 = (DIF - DEA) × 2
```

**判定规则**：

- **金叉**：前一时刻 DIF ≤ DEA 且当前 DIF > DEA
- **死叉**：前一时刻 DIF ≥ DEA 且当前 DIF < DEA
- **顶背离**：价格创新高但 DIF 未创新高
- **底背离**：价格创新低但 DIF 未创新低

**数据要求**：至少需要 35 根 K 线

**实现位置**：`src/utils/technicalIndicators.ts` 中的 `calculateMACD`、`isMACDGoldenCross`、`isMACDDeathCross`、`hasMACDDivergence`

---

#### 布林带（Bollinger Bands）

**含义**：由中轨（移动平均线）和上下轨（标准差带）组成的价格通道，用于判断价格相对位置。

**参数**：

- **上轨附近**：价格接近上轨（阈值 5%）
- **中轨附近**：价格接近中轨（阈值 5%）
- **下轨附近**：价格接近下轨（阈值 5%）

**计算公式**：

```
1. 中轨 = 20日简单移动平均（SMA20）
2. 标准差 = 20日收盘价的标准差
3. 上轨 = 中轨 + 2 × 标准差
4. 下轨 = 中轨 - 2 × 标准差
```

**判定规则**：

- **上轨附近**：收盘价 ≥ 上轨 × (1 - 5%)
- **中轨附近**：|收盘价 - 中轨| ≤ 带宽 × 5%
- **下轨附近**：收盘价 ≤ 下轨 × (1 + 5%)

其中带宽 = 上轨 - 下轨

**数据要求**：至少需要 20 根 K 线

**实现位置**：`src/utils/technicalIndicators.ts` 中的 `calculateBollingerBands`、`isNearUpperBand`、`isNearMiddleBand`、`isNearLowerBand`

### 2. K 线形态筛选

实现了 6 种经典 K 线形态识别，基于最新一根或最近几根 K 线进行判断。

#### 锤头线（Hammer）

**特征**：

- 实体小（小于范围的 1/3）
- 下影线长（≥ 实体的 2 倍）
- 上影线短（小于范围的 1/3）

**意义**：潜在的底部反转信号，表明下方有支撑

**数据要求**：1 根 K 线

---

#### 射击之星（Shooting Star）

**特征**：

- 实体小（小于范围的 1/3）
- 上影线长（≥ 实体的 2 倍）
- 下影线短（小于范围的 1/3）

**意义**：潜在的顶部反转信号，表明上方有压力

**数据要求**：1 根 K 线

---

#### 十字星（Doji）

**特征**：开盘价和收盘价非常接近（实体/范围 < 1%）

**意义**：市场犹豫不决，多空力量平衡

**数据要求**：1 根 K 线

---

#### 吞没形态（Engulfing）

**特征**：

- 需要两根 K 线
- 第二根 K 线的实体完全吞没第一根的实体
- 两根 K 线方向相反

**意义**：强烈的反转信号

**数据要求**：2 根 K 线

---

#### 早晨之星（Morning Star）

**特征**：

- 需要三根 K 线
- 第一根：大阴线
- 第二根：小实体（可以是十字星或小阴阳线）
- 第三根：大阳线，且收盘价进入第一根实体的一半以上

**意义**：底部反转信号

**数据要求**：3 根 K 线

---

#### 黄昏之星（Evening Star）

**特征**：

- 需要三根 K 线
- 第一根：大阳线
- 第二根：小实体
- 第三根：大阴线，且收盘价进入第一根实体的一半以下

**意义**：顶部反转信号

**数据要求**：3 根 K 线

**实现位置**：`src/utils/candlestickPatterns.ts`

---

### 3. 趋势形态筛选

实现了 5 种趋势形态识别，基于最近 20 根 K 线（可配置）进行判断。

#### 上升趋势（Uptrend）

**判断方法**：线性回归斜率为正且显著

**计算公式**：

```
1. 对最近 N 根 K 线的收盘价进行线性回归
2. 计算斜率 slope
3. 如果 slope > 平均价格 × 0.1%，则认为是上升趋势
```

**意义**：价格整体向上，多头占优

**数据要求**：至少 20 根 K 线

---

#### 下降趋势（Downtrend）

**判断方法**：线性回归斜率为负且显著

**计算公式**：

```
1. 对最近 N 根 K 线的收盘价进行线性回归
2. 计算斜率 slope
3. 如果 slope < -平均价格 × 0.1%，则认为是下降趋势
```

**意义**：价格整体向下，空头占优

**数据要求**：至少 20 根 K 线

---

#### 横盘整理（Sideways）

**判断方法**：价格波动幅度小于阈值（默认 5%）

**计算公式**：

```
1. 计算最近 N 根的最高价和最低价
2. 波动幅度 = (最高价 - 最低价) / 最低价 × 100%
3. 如果波动幅度 < 5%，则认为是横盘
```

**意义**：市场处于震荡状态，方向不明

**数据要求**：至少 20 根 K 线

---

#### 突破形态（Breakout）

**判断方法**：收盘价突破前期高点 2% 以上

**计算公式**：

```
1. 计算前 N 根（不含最后一根）的最高价 prevHigh
2. 如果最后一根收盘价 > prevHigh × 1.02 且收阳，则认为是突破
```

**意义**：向上突破阻力位，可能开启上涨行情

**数据要求**：至少 21 根 K 线

---

#### 跌破形态（Breakdown）

**判断方法**：收盘价跌破前期低点 2% 以上

**计算公式**：

```
1. 计算前 N 根（不含最后一根）的最低价 prevLow
2. 如果最后一根收盘价 < prevLow × 0.98 且收阴，则认为是跌破
```

**意义**：向下跌破支撑位，可能开启下跌行情

**数据要求**：至少 21 根 K 线

**实现位置**：`src/utils/trendPatterns.ts`

### 数据流

1. **用户交互层**：用户在筛选面板中设置筛选条件
2. **状态管理层**：React useState 管理筛选状态
3. **持久化层**：localStorage 保存用户偏好（键名：`opportunity_filter_prefs`）
4. **筛选引擎层**：Web Worker 执行实际筛选逻辑（`opportunityFilterWorker.ts`）
5. **结果展示层**：过滤后的数据显示在表格中

**关键流程**：

- 用户修改筛选条件 → 触发 300ms 防抖 → 发送消息到 Worker
- Worker 接收筛选参数 → 根据激活的筛选条件智能重算对应指标 → 执行筛选 → 返回结果
- 主线程接收结果 → 更新表格显示

---

### 性能优化

1. **Web Worker 并行计算**

   - 所有复杂的指标计算在后台线程执行，避免阻塞 UI
   - 使用 `requestId` 机制取消过期任务，保证结果一致性

2. **智能重算策略**

   - 只在相关筛选条件激活时才计算对应指标
   - 例如：未勾选 MACD 相关选项时，不计算 MACD

3. **防抖处理**

   - 筛选条件变化时 300ms 防抖，减少不必要的计算
   - 用户快速调整参数时不会频繁触发重算

4. **增量更新**

   - K 线数据变化时使用增量同步策略（`set-data-patch`）
   - 仅传输变化的数据，减少通信开销

5. **条数上限控制**
   - 轻量过滤后若超过 `MAX_OPPORTUNITY_KLINE_CACHE_ENTRIES`，优先保留缓存命中的标的
   - 避免 Worker 处理过多数据导致性能问题

---

### 筛选逻辑

所有新增筛选项遵循以下规则：

- **OR 逻辑（同类别内）**：同一类别内勾选多项时，满足任一条件即保留
  - 例如：同时勾选「金叉」和「死叉」，只要满足其中一个就通过
- **AND 逻辑（不同类别间）**：不同类别之间是 AND 关系，必须同时满足
  - 例如：同时启用「RSI 范围」和「MACD 金叉」，两个条件都必须满足
- **可选性**：未勾选任何项时不按该类筛选
  - 例如：MACD 三个选项都不勾选，则不按 MACD 筛选

**与其他筛选条件的关系**：

- 与横盘筛选：AND 关系
- 与趋势线筛选：AND 关系
- 与单日异动筛选：AND 关系
- 与基础数据筛选（价格、市值等）：AND 关系

## 使用方法

1. 打开机会分析页面
2. 点击「筛选条件」按钮打开右侧抽屉
3. 展开「技术指标与形态」面板
4. 设置所需的筛选条件：
   - **RSI 范围**：输入最小值和最大值（如：30 ~ 70）
   - **MACD 状态**：勾选金叉、死叉或背离（可多选）
   - **布林带位置**：勾选上轨、中轨或下轨附近（可多选）
   - **K 线形态**：勾选需要的形态（可多选）
   - **趋势形态**：勾选需要的趋势（可多选）
5. 筛选结果会实时更新（300ms 防抖）
6. 点击「一键分析」保存当前筛选条件到本地偏好

**提示**：

- 勾选多个同类选项时，满足任一即可（OR 逻辑）
- 不同类别之间需同时满足（AND 逻辑）
- 未勾选任何项时，该类不参与筛选

## 注意事项

### 1. 数据要求

某些指标需要足够的历史数据才能计算：

| 指标          | 最少 K 线数 | 说明           |
| ------------- | ----------- | -------------- |
| RSI           | 15 根       | 14 周期 + 1    |
| MACD          | 35 根       | 26 + 9         |
| 布林带        | 20 根       | 20 周期        |
| 吞没形态      | 2 根        | 需要两根 K 线  |
| 早晨/黄昏之星 | 3 根        | 需要三根 K 线  |
| 趋势形态      | 20 根       | 默认回溯 20 根 |
| 突破/跌破     | 21 根       | 20 + 1         |

**处理方式**：

- 数据不足时，对应指标返回 false 或 null
- 不会影响其他指标的筛选
- 建议在「一键分析」时获取足够的 K 线数据（默认 300 根）

---

### 2. 计算精度

- 所有指标计算都使用了标准的金融公式
- 在极端情况下可能存在微小误差（浮点数精度）
- 使用了 `EPSILON` 容差处理边界情况

---

### 3. 性能考虑

- **首次筛选**：当股票数量较多时，可能需要几秒时间
- **后续筛选**：由于 K 线数据已缓存，速度会更快
- **建议**：
  - 先使用基础筛选（价格、市值等）缩小范围
  - 再使用技术指标筛选精细筛选
  - 避免同时启用过多复杂指标

---

### 4. 与现有功能的集成

- **横盘筛选**：可同时启用，逻辑为 AND
- **趋势线筛选**：可同时启用，逻辑为 AND
- **单日异动**：可同时启用，逻辑为 AND
- **基础数据筛选**：在主线程先执行轻量过滤，再交给 Worker 处理复杂指标
