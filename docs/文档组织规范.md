## 📁 标准文档结构

所有功能模块**必须**遵循以下标准结构：

```
模块目录/
├── README.md                      # 模块索引（必须）
├── 01-使用指南.md                  # 面向用户（可选）
└── 02-技术实现.md                  # 面向开发者（可选）
```

### 结构说明

- **README.md** - 模块索引文档，必须包含

  - 模块简介
  - 文档导航（区分用户/开发者）
  - 快速索引表格
  - 功能概览
  - 使用建议
  - 更新记录

- **01-使用指南.md** - 面向普通用户

  - 功能介绍
  - 参数说明
  - 使用技巧
  - 常见问题

- **02-技术实现.md** - 面向开发者
  - 整体架构
  - 实现细节
  - 扩展方法
  - 性能优化
  - 修复验证
  - 技术特性

### 标准示例：机会分析模块

```
docs/机会分析/
├── README.md                      # ✅ 标准模块索引
├── 01-使用指南.md                  # ✅ 用户指南
└── 02-技术实现.md                  # ✅ 技术实现
```

## 🚫 禁止事项

### ❌ 不要创建总结文档

- ❌ `xxx总结.md`
- ❌ `xxx报告.md`
- ❌ `xxx完成总结.md`

**原因**：总结文档内容最终都应该整合到核心文档中，不应单独存在。

### ❌ 不要创建临时文档

- ❌ `xxx临时方案.md`
- ❌ `xxx草稿.md`
- ❌ `xxx笔记.md`

### ❌ 不要创建过细的文档

- ❌ `xxx样式调整.md`
- ❌ `xxx代码清理.md`
- ❌ `xxx优化说明.md`

**原因**：这些小改动应该记录在核心文档的"更新记录"中，而不是创建新文档。

## ✅ 新增功能时的文档流程

### 步骤 1: 确定所属模块

根据新功能所在的页面，确定属于哪个模块：

- 热门行情页面 → `docs/热门行情/`
- 行业板块页面 → `docs/行业板块/`
- 概念板块页面 → `docs/概念板块/`
- 提醒管理页面 → `docs/提醒管理/`
- 机会分析页面 → `docs/机会分析/`
- 数据概况页面 → `docs/数据概况/`

### 步骤 2: 创建或更新文档

#### 情况A：模块已有标准结构

- ✅ 用户功能 → 更新 `01-使用指南.md`
- ✅ 技术实现 → 更新 `02-技术实现.md`
- ✅ 更新记录 → 在对应文档底部添加

#### 情况B：模块还没有文档

1. 创建 `README.md`（参照标准结构）
2. 创建 `01-使用指南.md`（如需要）
3. 创建 `02-技术实现.md`（如需要）

### 步骤 3: 更新索引

1. 更新模块 `README.md` 中的快速索引表格
2. 更新 `docs/README.md` 中的对应模块部分
3. 添加更新记录

### 步骤 4: 检查清单

- [ ] 文档放在正确的模块目录下
- [ ] 模块 README.md 已更新
- [ ] 总索引 `docs/README.md` 已更新
- [ ] 所有链接都有效（相对路径）
- [ ] 文档格式统一
- [ ] 添加了更新记录
- [ ] 没有创建总结/报告类文档
- [ ] 相关功能整合到核心文档

## 📋 文档质量检查

在提交文档前，必须检查：

### 结构检查

- [ ] 模块有 `README.md` 索引
- [ ] 最多2个核心文档（01-使用指南 + 02-技术实现）
- [ ] 没有总结/报告/临时文档
- [ ] 文档数量控制在最小范围

### 内容检查

- [ ] README.md 包含所有必需章节
- [ ] 文档导航清晰（用户/开发者分离）
- [ ] 快速索引表格完整
- [ ] 功能概览使用树形图
- [ ] 更新记录详细准确

### 格式检查

- [ ] 标题层级正确（H1 → H2 → H3）
- [ ] 代码块有语言标注
- [ ] 表格对齐美观
- [ ] 使用 Emoji 提升可读性
- [ ] 链接使用相对路径

## 💡 最佳实践

1. **文档即代码**：像对待代码一样对待文档，保持整洁和一致性
2. **及时更新**：功能变更时同步更新文档
3. **用户导向**：优先编写用户需要的文档
4. **易于导航**：确保用户可以快速找到所需信息
5. **保持简洁**：一个模块最多2个核心文档
6. **整合优先**：相关内容整合到核心文档，不创建新文档
7. **更新记录**：每次更新都记录在底部
