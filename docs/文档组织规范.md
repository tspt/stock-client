## 📝 文档编写规范

### 1. 每个模块必须有 README.md

每个路由对应的文档目录都应该有一个 `README.md` 作为模块索引，包含：

- 模块简介
- 文档列表（带链接和简要说明）
- 适用人群
- 相关文档链接
- 最后更新时间

**示例：** 参见 [`docs/提醒管理/README.md`](./提醒管理/README.md)

### 2. 文档命名规范

- **用户指南类**：使用中文，清晰描述功能，如 `智能预警系统增强功能说明.md`
- **技术文档类**：可以使用英文或中文，如 `02-技术实现.md`
- **报告类**：明确标注类型，如 `智能预警系统逻辑检查报告.md`
- **序号前缀**：有阅读顺序的文档使用序号，如 `01-使用指南.md`、`02-技术实现.md`

### 3. 文档内容结构

每个文档应该包含：

```markdown
# 标题

## 功能概述

简要说明功能是什么

## 使用方法

详细的操作步骤

## 技术实现（可选）

面向开发者的技术细节

## 注意事项

重要提示和限制

## 相关文档

链接到相关文档
```

### 4. 交叉引用

- 使用相对路径链接其他文档
- 在 README 中提供完整的导航
- 在每个子文档顶部添加"返回上级"链接

### 5. 更新记录

在每个模块的 README 底部添加更新历史：

```markdown
## 🔄 更新历史

- **2026-04-17**: 新增智能预警系统文档
- **2026-04-10**: 创建文档索引系统
```

## 🔧 新增功能时的文档流程

### 步骤 1: 确定所属路由

根据新功能所在的页面，确定属于哪个路由：

- 热门行情页面 → `docs/热门行情/`
- 股票列表/详情页 → 代码注释
- 提醒管理页面 → `docs/提醒管理/`
- 数据概况页面 → `docs/optimizations/`
- 机会分析页面 → `docs/机会分析/`

### 步骤 2: 创建文档

1. 如果该路由目录下还没有 `README.md`，先创建它
2. 在对应目录下创建功能文档
3. 遵循文档命名规范

### 步骤 3: 更新索引

1. 在新建模块的 `README.md` 中添加新文档链接
2. 更新 `docs/README.md` 中的对应路由部分
3. 添加更新记录

### 步骤 4: 添加交叉引用

在其他相关文档中添加对新文档的引用链接。

## ✅ 文档质量检查清单

在提交文档前，请检查：

- [ ] 文档放在正确的路由目录下
- [ ] 模块 README.md 已更新
- [ ] 总索引 `docs/README.md` 已更新
- [ ] 所有链接都有效
- [ ] 文档格式统一（标题层级、列表样式等）
- [ ] 添加了更新记录
- [ ] 没有拼写错误
- [ ] 代码示例可以正常运行（如果有）

## 🚫 常见错误

1. ❌ 文档放在错误的目录下
2. ❌ 忘记更新 README.md 索引
3. ❌ 链接使用绝对路径（应该用相对路径）
4. ❌ 文档没有更新记录
5. ❌ 缺少模块级别的 README.md

## 💡 最佳实践

1. **文档即代码**：像对待代码一样对待文档，保持整洁和一致性
2. **及时更新**：功能变更时同步更新文档
3. **用户导向**：优先编写用户需要的文档
4. **易于导航**：确保用户可以快速找到所需信息
5. **保持简洁**：避免冗余，一个主题一个文档
