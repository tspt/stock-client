## 🔍 数据格式说明

### 股票数据文件格式 (JSON)

```json
{
  "data": {
    "code": "SH603938",
    "name": "三孚股份",
    "dailyLines": [
      {
        "time": 1738771200000,
        "open": 10.63,
        "close": 10.68,
        "high": 10.69,
        "low": 10.53,
        "volume": 18443
      }
      // ... 更多K线数据
    ]
  },
  "buypointDate": ["2025/12/22", "2026/04/01"]
}
```

**关键字段**:

- `data.code`: 股票代码
- `data.dailyLines`: K 线数据数组
- `buypointDate`: 买点日期数组（支持多种格式）

## ⚠️ 注意事项

### 1. 文件兼容性

- v4.0 脚本**仅支持** JSON 格式
- v3.0 脚本可能不支持新的 JSON 格式
- 两个版本可以并行使用，互不影响

### 2. 数据一致性

- 确保 `buypointDate` 中的日期在 K 线数据中存在
- 日期格式会自动标准化为 `YYYY-MM-DD`
- 无效日期会被跳过并记录警告

### 3. 性能考虑

- 69 只股票的数据量较大，训练时间可能较长
- 建议在有足够内存的环境中运行
- 可以考虑分批处理或优化参数

## 🐛 常见问题

### Q1: 提示"读取导出文件失败"

**A**: 需要先在历史回测页面导出数据：

1. 打开应用的历史回测页面
2. 执行全量回测
3. 点击"导出所有数据"
4. 文件会自动保存到 `历史回测数据/backtest_export_latest.json`

### Q2: 某些股票的买点未被识别

**A**: 可能原因：

- 买点日期不在 K 线数据范围内
- 日期格式不正确
- 模型尚未重新训练

### Q3: 如何回退到 v3.0？

**A**: 直接使用 v3.0 的脚本即可：

```bash
node verify-buypoint-coverage.cjs           # 验证
node v3.0_ml_buypoint_model_enhanced.cjs    # 训练
```
