# 股票 K 线数据导出功能 - 使用说明

## 功能概述

在历史回测页面，用户可以将选中股票的 K 线数据和自定义的日期买点保存到本地文件。

## 使用步骤

### 1. 打开历史回测页面

点击左侧导航栏的"历史回测"菜单项。

### 2. 选择股票

在左侧股票列表中，找到需要导出的股票。

### 3. 点击"复制"按钮

在每个股票项的右侧，点击"复制"按钮，会弹出导出抽屉。

### 4. 填写日期买点

在弹出的抽屉中：

- 股票名称和代码会自动填充（只读）
- 在文本框中输入日期买点，每行一个日期
- **只需输入 8 位数字**，例如：`20251222`
- 系统会自动格式化为：`2025/12/22`

示例输入：

```
20251222
20260401
20260515
```

### 5. 点击"保存"

点击"保存"按钮后：

- 系统会从 IndexedDB 获取该股票的完整 K 线数据
- 将数据保存为 JSON 格式到本地文件
- 文件路径：`docs/回测优化/stock_data/{股票名称}.txt`
- 如果文件已存在，会直接覆盖
- 保存成功后会显示文件路径

## 文件格式

保存的文件包含两部分：

1. **JSON 格式的 K 线数据**：

```json
{
    "code": "SH603938",
    "name": "三孚股份",
    "dailyLines": [...],
    "latestQuote": {...},
    "updatedAt": 1777530947177
}
```

2. **日期买点信息**（追加在 JSON 后面）：

```
日期买点：2025/12/22，2026/04/01，2026/05/15
```

## 注意事项

1. **日期格式**：必须输入 8 位数字，系统会自动验证日期的有效性

   - ✅ 正确：`20251222`
   - ❌ 错误：`2025-12-22`、`2025/12/22`、`2025122`

2. **多行输入**：每个日期占一行，空行会被自动忽略

3. **文件覆盖**：如果同名文件已存在，会直接覆盖，无需确认

4. **数据来源**：K 线数据来自 IndexedDB 的`stockHistory`表，确保该股票已有历史数据

5. **保存位置**：文件保存在项目目录下的 `docs/回测优化/stock_data/` 文件夹

## 技术实现

- **主进程**：`electron/main.ts` - 处理文件写入 IPC 请求
- **Preload**：`electron/preload.ts` - 暴露 saveStockData 方法
- **渲染进程**：`src/pages/BacktestPage/BacktestPage.tsx` - UI 交互和数据获取
- **类型定义**：`src/types/electron.d.ts` - TypeScript 类型声明

## 常见问题

### Q: 提示"未找到该股票的历史数据"？

A: 该股票尚未同步到 IndexedDB，请先在机会分析页面触发数据同步。

### Q: 提示"日期格式错误"？

A: 请检查输入的日期是否为 8 位数字，且是有效日期（如月份 1-12，日期 1-31）。

### Q: 保存的文件在哪里？

A: 在项目根目录下的 `docs/回测优化/stock_data/` 文件夹中。

### Q: 可以批量保存多个股票吗？

A: 当前版本仅支持单个股票保存，需要逐个操作。
