# 历史回测数据导出功能使用说明

## 功能概述

历史回测页面新增了"导出指定股票"功能，可以将 stock_data 目录中股票的回测结果导出为 JSON 格式，方便进行数据分析和验证。

**核心特性**：

- ✅ **自动扫描**：打开导出模态框时自动扫描 stock_data 目录
- ✅ **默认全选**：自动选中所有股票，可手动调整
- ✅ **固定路径**：导出文件保存到 `docs/回测优化/历史回测数据` 目录
- ✅ **单一格式**：仅支持 JSON 格式，适合程序化处理

## 使用步骤

### 1. 执行回测

首先在历史回测页面执行全量回测，生成回测数据。

### 2. 打开导出功能

点击页面顶部的“导出指定股票”按钮，系统会：

1. 自动打开导出模态框
2. 自动扫描`stock_data`目录获取最新股票列表
3. 默认选中所有股票

### 3. 调整股票选择（可选）

- 默认选中所有在`stock_data`目录中的股票
- 可以手动取消选中不需要导出的股票
- 支持全选/取消全选操作

### 4. 执行导出

点击“导出”按钮，系统将：

1. 从 IndexedDB 获取选中股票的回测数据
2. 转换为 JSON 格式
3. 保存到`docs/回测优化/历史回测数据`目录（固定路径）

## 导出文件格式

### JSON 格式（唯一格式）

```json
{
  "exportTime": "2026-05-06T10:00:00.000Z",
  "format": "json",
  "totalStocks": 28,
  "totalSignals": 150,
  "data": [
    {
      "code": "SH603989",
      "name": "艾华集团",
      "signals": [
        {
          "signalDate": "2025-01-01",
          "entryPrice": 15.73,
          "returns": {
            "day3": 2.5,
            "day5": 3.2,
            "day10": 5.1,
            "day20": 8.3
          }
        }
      ]
    }
  ]
}
```

**特点**：

- 完整保留数据结构
- 包含元数据（导出时间、股票数量、信号数量）
- 适合程序化处理和数据分析

## 更新股票列表

### 自动扫描（唯一方式）

每次打开导出模态框时，系统会：

1. 自动扫描`docs/回测优化/stock_data`目录
2. 提取所有 txt 文件中的股票代码和名称
3. 实时更新股票列表
4. 默认选中所有股票

**优点**：

- 完全自动化，无需任何操作
- 即时生效，始终保持最新状态
- 适合频繁添加新股票的场景

## 注意事项

1. **数据范围**：仅导出 stock_data 目录中的股票，非 IndexedDB 中的所有数据
2. **数据完整性**：只有执行过回测的股票才会有导出数据
3. **文件位置**：导出文件固定保存在`docs/回测优化/历史回测数据`目录
4. **文件格式**：仅支持 JSON 格式，适合程序化处理
5. **命名规则**：文件名包含时间戳，避免覆盖
6. **性能考虑**：大量股票导出可能需要较长时间
7. **降级方案**：如果 Electron API 不可用，数据会复制到剪贴板
8. **自动扫描**：每次打开导出模态框时自动扫描目录，无需手动刷新

## 故障排除

### 问题：导出按钮置灰

- 原因：没有回测数据
- 解决：先执行全量回测

### 问题：导出失败

- 检查是否有足够的磁盘空间
- 确认目录权限
- 查看控制台错误信息

### 问题：部分股票无数据

- 这些股票可能尚未执行回测
- 在回测页面确认股票有信号数据

## 技术实现

### 前端实现

- 使用 Ant Design Modal 组件提供用户界面
- 通过 IndexedDB 获取回测数据
- 支持 JSON 和 CSV 两种格式转换

### 后端实现

- 扩展 Electron IPC 处理器支持批量导出
- 复用现有的文件保存机制
- 自动创建必要的目录结构

### 配置管理

- 使用 TypeScript 配置文件管理股票列表
- 支持动态更新和自定义选择
- 提供工具脚本自动生成配置
