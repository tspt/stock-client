### 2. 性能指标说明

#### 精确率 (Precision) ⭐ **核心指标**
- **定义**: 预测为好信号的样本中，真正是好信号的比例
- **公式**: TP / (TP + FP)
- **意义**: 精确率越高，误报越少，信号质量越好
- **目标**: > 70%（越高越好）

#### 召回率 (Recall)
- **定义**: 所有真正的好信号中，被正确识别的比例
- **公式**: TP / (TP + FN)
- **意义**: 召回率越高，漏掉的好信号越少
- **权衡**: 提高精确率通常会降低召回率

#### F1分数
- **定义**: 精确率和召回率的调和平均
- **公式**: 2 * (Precision * Recall) / (Precision + Recall)
- **意义**: 综合评价指标

#### AUC-ROC
- **定义**: 模型区分能力的综合指标
- **范围**: 0.5-1.0（1.0为完美）
- **意义**: > 0.8表示模型有较好的区分能力

---

### 3. 混淆矩阵解读

```
              预测坏信号  预测好信号
   实际坏信号     TN          FP      ← FP是误报（坏信号被预测为好）
   实际好信号     FN          TP      ← FN是漏报（好信号被预测为坏）
```

- **TN (True Negative)**: 正确预测为坏信号
- **FP (False Positive)**: 误报（坏信号被预测为好）← **我们要最小化这个**
- **FN (False Negative)**: 漏报（好信号被预测为坏）
- **TP (True Positive)**: 正确预测为好信号

**精确率 = TP / (TP + FP)** ← FP越小，精确率越高

## ⚙️ 参数调整

如果需要调整模型行为，可以修改 `train_random_forest_model.py` 中的配置：

### 1. 提高精确率（更谨慎）

```python
PREDICTION_THRESHOLD = 0.7  # 提高到0.7，进一步减少误报
class_weight={0: 1, 1: 3}   # 增加好信号权重
```

**效果**: 精确率↑，召回率↓

### 2. 提高召回率（更激进）

```python
PREDICTION_THRESHOLD = 0.5  # 降低到0.5，捕捉更多好信号
class_weight={0: 1, 1: 1.5} # 降低好信号权重
```

**效果**: 召回率↑，精确率↓

### 3. 调整模型复杂度

```python
N_ESTIMATORS = 150      # 增加树的数量（提高稳定性）
MAX_DEPTH = 15          # 增加深度（提高拟合能力，但可能过拟合）
MIN_SAMPLES_SPLIT = 10  # 减少分裂所需样本（更敏感）
```

### 4. 调整采样比例

```python
TARGET_RATIO = 2.0  # 改为1:2（更平衡，但可能降低精确率）
TARGET_RATIO = 3.0  # 改为1:3（保留更多负样本，提高精确率）
```

## ❓ 常见问题

### Q1: 为什么有些信号的特征计算失败？

**原因**:
- 股票K线数据文件不存在
- 数据不足（少于60天）
- 信号日期在K线数据中找不到

**解决**: 这是正常现象，脚本会跳过这些信号。如果失败率过高（>20%），检查数据完整性。

---

### Q2: 精确率不够高怎么办？

**建议**:
1. 提高预测阈值（从0.6到0.7或0.8）
2. 增加好信号的类别权重（从2到3或4）
3. 增加负样本比例（TARGET_RATIO从2.5到3.0）
4. 检查特征工程，可能需要添加更多有效特征

---

### Q3: 模型文件大小太大怎么办？

**原因**: 100棵树的随机森林JSON文件约5-10MB

**解决方案**:
1. 减少树的数量（从100到50）
2. 限制树的深度（从12到8）
3. 使用方案B（ONNX格式）或方案C（单棵决策树）

---

### Q4: 如何重新训练模型？

**步骤**:
1. 如果有新的回测数据，先运行 `extract_and_label_signals.py`
2. 然后运行 `train_random_forest_model.py`
3. 生成的新模型文件会覆盖旧文件

---

### Q5: 如何在TypeScript中使用模型？

### 模型架构

- **算法**: 随机森林分类器 (RandomForestClassifier)
- **树数量**: 100
- **最大深度**: 12
- **节点最少样本数**: 15
- **叶子最少样本数**: 8
- **类别权重**: {坏信号: 1, 好信号: 2}
- **预测阈值**: 0.6（可调）

---

### 数据划分策略

- **方法**: GroupShuffleSplit（按股票分组）
- **训练集**: 70%
- **测试集**: 30%
- **目的**: 防止同一股票的数据同时出现在训练和测试集中，避免数据泄露
