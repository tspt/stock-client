# 回测优化模块

本模块包含股票买点识别的机器学习模型及相关验证工具。

---

## 📁 目录结构

### 核心目录

- **[模型训练/](./模型训练/)** - 机器学习模型训练脚本（v3.0 和 v4.0）
- **[股票数据/](./股票数据/)** - 69 只股票的 K 线数据和买点标记
- **[验证脚本/](./验证脚本/)** - 覆盖率验证和分析工具（v3.0 和 v4.0）
- **[买点验证报告/](./买点验证报告/)** - 验证结果和详细分析报告
- **[历史回测数据/](./历史回测数据/)** - 回测导出文件
- **[文档归档/](./文档归档/)** - 历史文档（已清空）

## 🚀 快速开始

### 📌 版本选择

**v3.0 (稳定版)**: 适用于 28 只股票、78 个买点的数据集
**v4.0 (最新版)**: 适用于 69 只股票、188 个买点的扩展数据集 ⭐ 推荐

### 1️⃣ 训练模型

#### 使用 v4.0 (推荐) ⭐

```bash
cd 模型训练
node v4.0_ml_buypoint_model_enhanced.cjs
```

**输出**: 生成`buypoint_model.json`，需复制到`src/utils/analysis/buypoint_model_v4.json`

#### 使用 v3.0 (历史版本)

```bash
cd 模型训练
node v3.0_ml_buypoint_model_enhanced.cjs
```

**注意**: v3.0 适用于 28 只股票、78 个买点的数据集，已被 v4.0 替代

### 2️⃣ 添加新样本

1. 将新股票的 K 线数据（JSON 格式）放入`股票数据/`目录
2. 在 JSON 中添加`buypointDate`字段标记买点：`"buypointDate": "YYYY-MM-DD"`
3. 编辑`模型训练/v4.0_ml_buypoint_model_enhanced.cjs`中的股票列表配置
4. 重新训练模型并部署到应用

### 3️⃣ 验证买点覆盖率

#### 使用 v4.0 (推荐) ⭐

```bash
cd 验证脚本
node verify-buypoint-coverage.v4.0.cjs      # 验证覆盖率
node analyze-missed-buypoints.v4.0.cjs      # 分析未覆盖买点特征
node compare-covered-vs-missed.v4.0.cjs     # 对比已覆盖vs未覆盖
```

#### 使用 v3.0 (历史版本)

```bash
cd 验证脚本
node verify-buypoint-coverage.v3.0.cjs      # v3.0覆盖率验证
node analyze-missed-buypoints.v3.0.cjs      # v3.0未覆盖分析
node compare-covered-vs-missed.v3.0.cjs     # v3.0对比分析
```

### 4️⃣ 导出回测数据

1. 在应用的"历史回测"页面执行全景回测
2. 点击"导出数据"按钮
3. 文件自动保存到`历史回测数据/backtest_export_latest.json`
4. 运行验证脚本检查覆盖率

## 📊 当前状态

### 模型版本

- **当前模型**: v4.0 Config_22 (稳定版) ⭐
- **配置参数**: maxDepth=20, minSamplesLeaf=1, negativeSamples=25/股票
- **数据集**: 69 只股票，188 个手动标记买点
- **集成策略**: 单决策树模型（非集成学习）

### 性能指标

| 指标           | 数值       | 说明           |
| -------------- | ---------- | -------------- |
| **训练召回率** | 80.85%     | 训练集表现     |
| **真实覆盖率** | **78.19%** | 实际回测验证   |
| **精确率**     | 100.00%    | 无误报         |
| **F1 分数**    | 0.89       | 综合性能       |
| **已覆盖买点** | 147 个     | 188 个中的 147 |
| **未覆盖买点** | 41 个      | 待优化空间     |

### 关键发现

- ✅ Config_22 是当前最优选择，平衡了召回率和泛化能力
- ❌ Config_28(minSamplesLeaf=2)严重过拟合：训练集 99.5%，真实仅 64.36%
- ❌ Config_30/33 过于保守：覆盖率分别降至 22.87%和 5.85%
- 💡 增加 minSamplesLeaf 会导致模型过于保守，不适合当前数据集
