# 集成学习探索总结报告

**日期**: 2026-05-08  
**目标**: 通过集成学习将买点模型覆盖率从 78.19%提升至 80-85%  
**结果**: ❌ 集成学习方案未成功，Config_22 仍为最优选择

---

## 📊 测试概况

### 测试的配置

| 配置      | minSamplesLeaf | negativeSamples/股票 | 训练集召回率 | 真实覆盖率 | 结论        |
| --------- | -------------- | -------------------- | ------------ | ---------- | ----------- |
| Config_22 | 1              | 25                   | 80.85%       | **78.19%** | ✅ **最优** |
| Config_30 | 2              | 30                   | 未知         | 22.87%     | ❌ 过低     |
| Config_33 | 3              | 30                   | 89.9%        | 5.85%      | ❌ 极低     |

### 关键发现

1. **minSamplesLeaf 参数对召回率影响巨大**

   - minSamplesLeaf=1: 覆盖率 78.19%（基准）
   - minSamplesLeaf=2: 覆盖率降至 22.87%（下降 55 个百分点）
   - minSamplesLeaf=3: 覆盖率降至 5.85%（下降 72 个百分点）

2. **增加正则化会导致模型过于保守**

   - Config_30 和 Config_33 几乎不产生任何信号
   - 这与预期相反（预期会获得互补的决策边界）

3. **Config_22 是目前的最优选择**
   - 在召回率和精确率之间取得了最佳平衡
   - 训练集召回率 80.85%，真实覆盖率 78.19%，泛化能力良好

---

## 🔍 问题分析

### 为什么集成学习失败？

1. **缺乏真正的模型多样性**

   - Config_22、Config_30、Config_33 使用相同的特征和算法
   - 仅通过调整 minSamplesLeaf 无法产生足够差异化的决策边界
   - OR 逻辑集成需要的是"不同视角"的模型，而非"不同严格程度"的模型

2. **数据集特性限制**

   - 188 个买点分布在 69 只股票上，平均每只股票 2.7 个买点
   - 数据量相对较小，难以支撑多个差异化模型的训练
   - minSamplesLeaf>1 会导致大量叶子节点无法满足最小样本要求

3. **决策树的本质限制**
   - 单棵决策树已经能够捕捉主要模式
   - 进一步的正则化只会减少信号，不会增加新的识别能力

## 📝 技术细节

### Config_33 训练失败的修复

**问题**：`train-config33-quick.cjs`使用错误的字段名`d.date`匹配买点日期

**修复**：

```javascript
// 修复前
const idx = klineData.findIndex((d) => d.date === bpDate);

// 修复后
const normalizedBpDate = bpDate.replace(/\//g, '-');
const idx = klineData.findIndex((d) => {
  const date = new Date(d.time);
  const dateStr = `${date.getFullYear()}-${String(date.getMonth() + 1).padStart(2, '0')}-${String(
    date.getDate()
  ).padStart(2, '0')}`;
  return dateStr === normalizedBpDate;
});
```

**结果**：Config_33 成功训练，正样本从 0 个恢复到 187 个
