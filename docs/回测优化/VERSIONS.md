## 📊 v4.0 模型系列 (2026-05-08)

### 训练配置对比

| 配置 ID   | maxDepth | minSamplesLeaf | negativeSamples | 训练集召回率 | 真实覆盖率 | 状态      |
| --------- | -------- | -------------- | --------------- | ------------ | ---------- | --------- |
| Config_22 | 20       | 1              | 25/股票         | 80.85%       | **78.19%** | ✅ 稳定版 |
| Config_28 | 20       | 2              | 20/股票         | 99.5%        | 64.36%     | ⚠️ 过拟合 |
| Config_30 | 20       | 2              | 30/股票         | 未知         | 22.87%     | ❌ 过低   |
| Config_33 | 20       | 3              | 30/股票         | 89.9%        | 5.85%      | ❌ 极低   |

**当前使用**: Config_22（稳定版）

### 关键发现

- **Config_28 过拟合问题**: 训练集召回率 99.5%，但真实回测仅 64.36%，差距 35 个百分点
- **Config_30/33 过于保守**: 增加 minSamplesLeaf 导致召回率大幅下降（22.87%和 5.85%）
- **根本原因**: minSamplesLeaf>1 会导致模型过于保守，无法识别足够的买点
- **最佳实践**: Config_22 的 minSamplesLeaf=1 是当前最优选择，平衡了召回率和精确率

## 🔍 验证脚本版本

### v4.0 系列 (2026-05-08)

#### verify-buypoint-coverage.v4.0.cjs

- **版本**: v4.0
- **功能**: 验证 v4.0 模型买点覆盖率
- **创建时间**: 2026-05-08
- **主要特性**:
  - 读取 69 只股票的 JSON 数据文件
  - 提取 buypointDate 字段中的手动标记日期
  - 对比回测导出文件中的模型信号日期
  - 统计覆盖率和未覆盖买点列表
- **使用场景**: 每次模型更新后验证覆盖率

#### analyze-missed-buypoints.v4.0.cjs

- **版本**: v4.0
- **功能**: 分析 v4.0 模型未覆盖买点的共同特征
- **创建时间**: 2026-05-08
- **主要特性**:

  - 统计分析未覆盖买点的价格形态、成交量、均线关系
  - 聚类分析：缩量整理型、小幅波动型、均线突破型、其他类型
- **使用场景**: 找出未覆盖买点的共性，指导模型优化

#### compare-covered-vs-missed.v4.0.cjs

- **版本**: v4.0
- **功能**: 对比 v4.0 已覆盖和未覆盖买点的特征差异
- **创建时间**: 2026-05-08
- **主要特性**:
  - 提取已覆盖和未覆盖买点的技术特征
  - 统计分析两组样本的差异
  - 生成特征对比报告
- **使用场景**: 理解模型为什么拒绝某些买点

#### verify-missed-points-characteristics.v1.0.cjs

- **版本**: v1.0

- **使用场景**: 在调整模型参数前验证假设

#### analyze-model-decision-tree.v1.0.cjs

- **版本**: v1.0

## 🔄 版本演进路线

### v1.0 → v2.0 → v3.0 → v4.0

**v1.0-v2.0**: 规则模型阶段

- 基于手工设计的规则
- 可解释性强，但灵活性差

**v3.0**: 集成学习阶段

- 多个决策树投票（OR 逻辑）
- 召回率高，但复杂度高

**v4.0**: 单决策树优化阶段

- 单一决策树，简化架构
- 系统性参数搜索
- 当前稳定版：Config_22

### 未来方向

**v4.1** (计划中):

- 尝试更强的正则化（minSamplesLeaf=3-5）
- 降低树深度至 12-15
- 目标：真实覆盖率 ≥ 85%

**v5.0** (探索中):

- 随机森林或梯度提升树
- 深度学习（如果数据量足够）
- 目标：真实覆盖率 ≥ 90%

## 💡 使用建议

### 选择合适的版本

1. **生产环境**: 使用 `verify-buypoint-coverage.v4.0.cjs` + Config_22 模型（覆盖率 78.19%）
2. **模型优化**: 使用 `analyze-missed-buypoints.v4.0.cjs` + `analyze-model-decision-tree.v1.0.cjs`
3. **历史对比**: 使用 v3.0 系列脚本（仅供参考）
4. **集成学习探索**: 当前不建议，Config_30/33 表现不佳，需寻找其他差异化方案

### 运行脚本

```bash
# 验证覆盖率
node verify-buypoint-coverage.v4.0.cjs

# 分析未覆盖买点
node analyze-missed-buypoints.v4.0.cjs

# 分析决策树
node analyze-model-decision-tree.v1.0.cjs

# 验证特征假设
node verify-missed-points-characteristics.v1.0.cjs
```
