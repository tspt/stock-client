# 机器学习买点识别模型

## 📁 文件夹说明

本文件夹包含基于决策树的股票买点识别模型及相关分析工具。

---

## 📂 文件清单

### 核心文件

- **`ml_buypoint_model.cjs`** - 主程序:训练决策树模型
- **`buypoint_model.json`** - 训练好的模型文件

### 分析工具

- **`analyze_buypoint_features.cjs`** - 方向 1:买点特征全面分析
- **`clustering_analysis.cjs`** - 方向 2:买点分类聚类分析
- **`time_series_analysis.cjs`** - 方向 3:时间序列模式分析
- **`non_buypoint_comparison.cjs`** - 方向 4:对比非买点时期
- **`rule_combination_optimization.cjs`** - 方向 5:规则组合优化

---

## 🚀 使用方法

### 1. 训练模型

```bash
node ml_buypoint_model.cjs
```

**功能:**

- 从 6 只股票的 14 个买点样本中提取特征
- 随机选取 120 个非买点作为负样本
- 训练决策树分类模型
- 输出模型性能指标
- 保存模型到 `buypoint_model.json`

**输出示例:**

```
正样本(买点): 14个
负样本(非买点): 120个
总样本数: 134个

【性能指标】
准确率 (Accuracy): 94.8%
精确率 (Precision): 100.0%
召回率 (Recall): 36.4%
F1分数: 53.33
```

### 2. 运行分析工具

```bash
# 特征分析
node analyze_buypoint_features.cjs

# 聚类分析
node clustering_analysis.cjs

# 时间序列分析
node time_series_analysis.cjs

# 非买点对比
node non_buypoint_comparison.cjs

# 规则优化
node rule_combination_optimization.cjs
```

## 📊 当前模型性能

### 训练数据

- **正样本**: 14 个买点

  - 中衡设计: 3 个
  - 起帆股份: 3 个
  - 永杉锂业: 2 个
  - 山东玻纤: 2 个
  - 宏昌电子: 2 个
  - 三孚股份: 2 个

- **负样本**: 120 个随机非买点

### 模型指标

| 指标    | 数值   | 说明                           |
| ------- | ------ | ------------------------------ |
| 准确率  | 94.8%  | 整体预测准确度                 |
| 精确率  | 100.0% | 预测为买点的都是真买点(无误报) |
| 召回率  | 36.4%  | 捕获了 4/11 个买点             |
| F1 分数 | 53.33  | 综合评分                       |

### 决策规则

模型学习到的关键判断条件:

1. **距高点%** ≤ -26.08 → 买点 (100%置信度)
2. **布林带宽度** > 0.13 且 **ATR%** ≤ 3.05 → 进一步判断
3. **MA5 偏离%** 作为辅助判断条件

## 🔄 如何增加新样本

### 步骤 1: 收集新买点

当您发现新的优质买点时,记录:

- 股票代码
- 买入日期
- 次日涨幅(验证是否为有效买点)

### 步骤 2: 准备数据文件

将股票 K 线数据保存为 `{股票名称}.txt` 格式,放在 `../stock_data/` 目录下。

### 步骤 3: 更新配置

编辑 `ml_buypoint_model.cjs` 中的 `stockFiles` 数组:

```javascript
const stockFiles = [
  { file: '中衡设计.txt', dates: ['2026-04-17', '2026-01-07', '2025-11-24'] },
  { file: '起帆股份.txt', dates: ['2026-04-14', '2026-01-16', '2025-12-11'] },
  // ... 添加新股票
  { file: '新股票.txt', dates: ['2026-XX-XX'] },
];
```

### 步骤 4: 重新训练

```bash
node ml_buypoint_model.cjs
```

模型会自动使用新数据重新训练,生成更新的 `buypoint_model.json`。

## 💡 后续集成计划

当样本量达到 **50-100 个买点** 时,将模型集成到项目回测系统中:

### 集成步骤

1. **加载模型**: 在回测 worker 中读取 `buypoint_model.json`
2. **特征提取**: 对每只股票实时计算 9 个特征
3. **预测判断**: 调用模型预测是否为潜在买点
4. **结果展示**: 在回测结果中标记 AI 识别的买点

### 预期效果

- 自动化识别类似历史买点的机会
- 减少人工筛选工作量
- 随着样本增加,准确率持续提升

---

## 📝 注意事项

1. **样本质量**: 确保新增的买点确实是优质的(次日涨幅>3%)
2. **数据完整性**: K 线数据至少需要 60 个交易日
3. **模型局限**: 当前样本量小,可能存在过拟合,需谨慎使用
4. **定期更新**: 建议每增加 10 个新样本就重新训练一次
