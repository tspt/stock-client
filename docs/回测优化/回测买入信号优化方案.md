export function runBacktestScreening(klineData: KLineData[], index: number): boolean {
  if (index < 30) return false;

## 🎯 新方案设计

### 核心买入信号识别策略

**触发条件**：满足以下任意 **2 个或以上** 条件即发出买入信号

#### 信号指标清单

1. **RSI 超卖信号**

   - RSI(6) < 30 或 RSI(12) < 30
   - 原理：相对强弱指标进入超卖区，预示反弹机会

2. **MACD 金叉**

   - DIF 从下向上穿越 DEA
   - 原理：动量指标显示趋势转强

3. **接近布林带下轨**

   - 价格位置 < 30% 或 距下轨距离 < 5%
   - 原理：价格处于通道底部，有回归中轨需求

4. **接近 MA20 支撑**

   - 价格低于 MA20 且距离 < 5%
   - 原理：20 日均线是重要支撑位

5. **成交量萎缩**
   - 量比 < 0.8（当日成交量 / 5 日均量）
   - 原理：抛压减轻，筹码稳定

#### Step 2: 实现信号计算逻辑

```typescript
interface BuySignals {
  rsiOversold: boolean; // RSI超卖
  macdGoldenCross: boolean; // MACD金叉
  nearBollingerLower: boolean; // 接近布林带下轨
  nearMA20: boolean; // 接近MA20
  volumeShrink: boolean; // 成交量萎缩
}

function calculateBuySignals(klineData: KLineData[]): BuySignals {
  // 1. 计算RSI
  const rsi6 = calculateRSI(klineData, 6);
  const rsi12 = calculateRSI(klineData, 12);
  const rsiOversold = (rsi6 !== null && rsi6 < 30) || (rsi12 !== null && rsi12 < 30);

  // 2. 计算MACD并判断金叉
  const macd = calculateMACD(klineData);
  const macdGoldenCross = isMACDGoldenCross(macd.dif, macd.dea, klineData.length - 1);

  // 3. 计算布林带并判断位置
  const bb = calculateBollingerBands(klineData, 20, 2);
  const lastClose = klineData[klineData.length - 1].close;
  const nearBollingerLower = checkNearLowerBand(lastClose, bb);

  // 4. 计算MA20并判断距离
  const ma20 = calculateMA(klineData, 20);
  const nearMA20 = checkNearMA20(lastClose, ma20);

  // 5. 计算成交量比率
  const volumeShrink = checkVolumeShrink(klineData);

  return {
    rsiOversold,
    macdGoldenCross,
    nearBollingerLower,
    nearMA20,
    volumeShrink,
  };
}
```

#### Step 3: 实现辅助判断函数

```typescript
function checkNearLowerBand(close: number, bb: BollingerBands): boolean {
  if (!bb.upper || !bb.lower) return false;
  const bandwidth = bb.upper - bb.lower;
  if (bandwidth <= 0) return false;
  const position = (close - bb.lower) / bandwidth;
  return position < 0.3 || close <= bb.lower * 1.05;
}

function checkNearMA20(close: number, ma20: number): boolean {
  if (!ma20) return false;
  return close < ma20 && Math.abs(close - ma20) / ma20 < 0.05;
}

function checkVolumeShrink(klineData: KLineData[]): boolean {
  if (klineData.length < 5) return false;
  const currentVol = klineData[klineData.length - 1].volume;
  const avgVol5 = klineData.slice(-5).reduce((s, k) => s + k.volume, 0) / 5;
  return currentVol < avgVol5 * 0.8;
}
```

## ⚠️ 注意事项

### 边界情况处理

1. **数据不足**：当 K 线数据少于 30 根时，直接返回 false
2. **指标计算失败**：某个指标计算失败不应影响其他指标的判断
3. **空值处理**：所有指标返回值都需要进行 null 检查

### 性能考虑

1. **切片操作**：每次只传入截止到当前日期的数据切片
2. **重复计算**：可以考虑缓存已计算的指标值（如果需要优化性能）

## 💡 总结

本次优化的核心思想是：

> **从"形态确认"转向"底部识别"**

通过多个技术指标的综合判断，在股价到达底部区域时及时发出买入信号，而不是等待形态完全确认后再行动。这样可以在保证准确率的前提下，显著提高信号的及时性。

**关键技术点**：

- RSI 超卖捕捉极端情绪
- MACD 金叉确认趋势转折
- 布林带下轨识别价格低位
- MA20 支撑判断中期趋势
- 成交量萎缩验证抛压减轻

这套组合拳能够覆盖不同类型的底部形态，提高买入信号的覆盖率和准确性。

## ✅ 新方案: 基于决策树的机器学习模型

### 为什么选择决策树?

1. **可解释性强**: 可以清楚看到每个判断条件
2. **小样本友好**: 即使只有 14 个样本也能训练
3. **易于更新**: 新增样本后可以重新训练
4. **输出清晰**: 直接给出"是/否"的判断

---

### 模型训练结果

**训练数据**:

- 正样本: 14 个买点
- 负样本: 120 个随机非买点
- 总样本: 134 个

**模型性能**:
| 指标 | 数值 | 说明 |
|------|------|------|
| 准确率 | 94.8% | 整体预测准确度 |
| 精确率 | 100.0% | 预测为买点的都是真买点(无误报!) |
| 召回率 | 36.4% | 捕获了 4/11 个买点 |
| F1 分数 | 53.33 | 综合评分 |

**决策树规则**:

```
if 距高点% <= -26.08:
    → 买点 (100%置信度)

else if 布林带宽 <= 0.13:
    → 非买点

else if ATR% <= 3.05:
    if 距高点% <= -7.82:
        if 样本数=2: → 非买点 (50%)
        else: → 买点 (100%)
    else:
        if MA5偏离% <= 0.11:
            → 非买点 (19.4%)
        else:
            → 非买点
```

**关键特征重要性**:

1. 距高点% (使用 2 次) - 最重要的特征

## 💡 核心洞察总结

### 为什么传统技术指标失效?

1. **买点类型多样**: V 型反转、平台突破、回调企稳等多种形态并存
2. **不是超跌反弹**: 买点大多处于上升趋势中的小幅回调,而非深度暴跌
3. **综合判断**: 用户的买点识别是基于多维度综合判断,而非单一指标
4. **时间窗口**: 这是一个中期持有策略(10 天平均+18.7%),需要不同的识别逻辑

### 机器学习方案的优势

1. **自动学习**: 从历史数据中自动发现规律,无需人工设定规则
2. **可扩展性**: 样本越多,模型越准确
3. **可解释性**: 决策树可以清楚展示判断逻辑
4. **灵活性**: 可以轻松添加新特征(如板块热度、大盘走势等)

## 💡 关键发现

### 1. 树深度的影响

- **深度 4-5**: 召回率低（11.8%-23.5%），模型过于简单，无法学习复杂模式
- **深度 6**: 召回率提升到 52.9%，开始有效识别买点
- **深度 7**: 召回率进一步提升到 70.6%-73.5%
- **深度 8**: **最佳平衡点**，召回率 85.3%，精确率 100%

**结论**：增加树深度可以显著提升召回率，但需要配合适当的负样本数量。

### 2. 负样本数量的影响

- **过多负样本**（340 个，Config_5）：模型偏向预测"非买点"，召回率下降至 35.3%
- **适中负样本**（170-204 个）：最佳平衡，召回率高且精确率好
- **过少负样本**（170 个，Config_1/3/6）：精确率略有下降

**结论**：负样本与正样本的比例应控制在 **4:1 到 5:1** 之间。

### 3. 最优组合

**maxDepth = 8** + **负样本/股票 = 12** = **最佳配置**

- 既保证了足够的学习能力（深度 8）
- 又避免了过度偏向负样本（12 个/股票）
- 实现了召回率和精确率的双重提升
